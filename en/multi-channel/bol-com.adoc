== Registering with bol.com

Before setting up bol.com in plentymarkets you have to register with **link:https://bol.com.[bol.com, window="_blank"]** first.

[#200]
== Setting up bol.com in plentymarkets

Set up bol.com in plentymarkets to sell items on bol.com. To do so, proceed as follows.

[.instruction]
Setting up bol.com in plentymarkets

. Go to **Settings » Markets » bol.com » Basic settings**.
. Carry out the settings. Pay attention to the information given in table 1.
. **Save** the settings.

.Basic settings for the market bol.com
[cols="a,a"]
|====
|Setting |Explanation

|**Activate the automatic data exchange**
|If the data exchange is activated, then there will be a daily item export, an hourly stock synchronisation and an hourly order import.

|**Public key**
|Enter the API key as received from bol.com.

|**Private key**
|Enter the API key as received from bol.com.
|====

Connecting to this market allows for the following automatic processes to take place:

* Order import every 20 minutes

== Activating the order referrer

In order to link items, properties etc. with bol.com you will have to activate the order referrer bol.com in the menu **Settings » Orders » Order referrer**.

[.instruction]
Activating the order referrer bol.com:

. Go to **Settings » Orders » Order referrer**.
. Place a check mark next to **bol.com**.
. **Save** the settings.

== Setting the item availability

Items have to be available for bol.com. This is done in the **Availability** tab of an item within the <<item/managing-items#170, **Item » Edit item » Tab: Variation ID** >> menu.

[.instruction]
Setting the item availability for bol.com:

. Open the **Item » Edit item » Open item » Tab: Variation ID » Tab: Settings** menu.
. Activate the main variation in the **Availability** section.
. Click on the **Availability** tab.
. Click in the **Markets** section in the selection field. +
→ A list with all available markets will be displayed.
. Activate the option **bol.com**.
. Click on **Add**. +
→ The Market will be added.
. **Save** the settings. +
→ The item is available on bol.com.

The availability for variations can be individually edit in the **Item » Edit item » Open item » Tab: Variation ID » Open variation » Tab: Availability** menu.

Note that item descriptions have to be in English for bol.com.

== Linking categories

Link your store categories to the bol.com categories in order to display your items in these categories. Further items from the linked category will be assigned automatically.

[.instruction]
Linking categories:

. Go to **Settings » Markets » bol.com » Category link**.
. Click on **Search**.  +
→ The **Select category** window will open.
. Select the bol.com category that best matches your online store category.
. Click on **Apply**. +
→ The name and category path of the bol.com category will be displayed.
. If you know the name of the bol.com category, enter it directly into the **Market category** field to link it to your online store category.
. **Save** the settings.

== Creating properties

Next create <<item/managing-items#480, **properties** >> and link these with bol.com properties.

[.instruction]
Creating a property:

. Go to **Settings » Item » Properties**.
. Click on the **New property** tab.
. Enter a name for the property under **Back end name**.
. Select **nl** for Dutch from the **Language** drop-down menu.
. Enter the Dutch name that you want to use for this property into the **Name** field.
. Select the option **Text** from the **Property type** drop-down menu.
. Enter the bol.com property into the line **bol.com property**.
. **Save** the settings.

== Setting up the item and offer export

This chapter explains how to export your items and offers to bol.com. The elastic Export is used to transfer data to bol.com. First, create the data formats BolItemExport and BolOfferExport and than send the CSV file to bol.com. Both files have to be send in order to sell items on bol.com.

The following fields are necessary for the item export:

* EAN
* SKU
* Name
* Categories
* Item description
* Image
* Manufacturer

The following fields are necessary for the offer export:

* EAN
* SKU
* Fulfillment by
* Stock
* Price
* Item condition
* Shipping availability

[.instruction]
Setting up the export:

. Go to **Data exchange » Export » Elastic export**.
. Click on **New export**.
. Carry out the settings. Pay attention to the information given in table 2.
. Carry out further settings for item filters and format settings. Pay attention to the information given in table 5 in the <<data-exchange/exporting-data#60, **Exporting data** >> page of the manual.
. **Save** the settings. +
→ The export format will be given an ID and it will appear in the overview within the **Exports** tab.

.Settings for the item export
[cols="a,a"]
|====
|Setting |Explanation

2+|Settings

|**Name**
|Enter a name. The export format will be listed under this name in the overview within the **Exports** tab.

|**Type**
|Select a type from the drop-down menu. Currently, **Item** is the only available option. Further options will be available soon.

|**Format**
|Select the format **BolItemExport** for the item export or **BolOfferExport** for the offer export.

|**Limit**
|Enter a number. If you want to transfer more than 10,000 data records to bol.com, then the output file will not be generated again for another 24 hours. This is to save resources.

|**Provisioning**
|Select **FTP**.

|**File name**, +
**Server**, +
**Port**, +
**User name**, +
**Password**, +
**Path**
|Enter parameter. The access data including the password and user name are provided by bol.com. The standard FTP port is **21**. The BolOfferExport's file name has the structure &lt;SellerID&gt;.csv. The file has to be saved in the folder **Inbound**.

|**Transfer**
|Select **Daily** for the item export as bol.com allows a maximum of 4 export per day. **Hourly** for the offer export.
|====

== Viewing the API log

The menu **<<data-exchange/api-log#, Data exchange » API log  >>** contains a history of processes that run via the bol.com interface.

[.instruction]
Viewing the API log:

.  Go to **Data exchange » API log » Tab: API log**.
.  Select the **bol.com** format from the **Process** drop-down menu.
.  Select a **Date** if you only want to see entries for a specific day.
.  Click on **Search** to carry out the search. +
→ The processes will be displayed.
