== eKomi

Before potential customers decide to buy they want to know what the delivery time is and whether the product will be delivered after payment. Companies can increase their trust through certificates and awards. The most efficient way is through reflection of customer reviews, the link:http://www.ekomi-us.com[eKomi^] *5-star review system* helps increase the conversion ratio (Info: eKomi website; www.ekomi-us.com).

[IMPORTANT]
.Evaluations
====
The *evaluations* will be retrieved *once a day*.
====

[WARNING]
.Pay attention to the market's terms and conditions
====
Understand the particular market's terms and conditions before you set up a function in your online store that requests evaluations. Some markets only allow you to request evaluations if specific requirements are met. Some markets do not permit this at all. Open the *Procedure manager* and use the *Referrer* option to exclude the particular market from the automatic reminder function.
====

== Registering with eKomi

Click on the following link in order to register: link:http://www.ekomi.co.uk/uk/register[http://www.ekomi.co.uk/uk/register^]. Afterwards, you will receive the access data that you need for setting up the tool in plentymarkets.

== Setting up eKomi in plentymarkets

Read the explanations below on how to set up *eKomi* in your plentymarkets system.

=== Setting up a user

Your plentymarkets system will be connected to *eKomi* via the SOAP interface. In order for it to have access, this evaluation tool requires its own <<basics/admin-tasks/manage-users#, user account>> with access rights to the API in your system.

[.instruction]
Setting up a user for eKomi:

. Go to *System » System settings » Settings » User*.
. Set up a new user account for eKomi and assign it to the *variable* user class.
. Click on *Rights » API » Customer* and activate the *GetCustomers* call for this user. Then click on *Order* and activate the *SearchOrders* call.
. *Save* the settings.

[WARNING]
.Calls
====
Make sure that you *only activate these specific calls* for this user.
====

=== Sending data to eKomi

Afterwards, send an email to eKomi that contains the following information:

*API URL* +
e.g.: http://www.example.com/plenty/api/soap/version114/?xml

*The user name that was set up for the user account* +
e.g.: eKomi

*The password that was set up for the user account* +
e.g.: 123abc!:_ (Be sure to create your own password!)

Send this information to support@ekomi.de and put your eKomi ID or customer ID in the subject line.

=== Configuring the menu

Go to *eKomi* (see path above) and configure the settings as described below.

==== Tab: Add language

You can set up the corresponding *language* in this tab. You need to set up a language before the eKomi settings become available.

== Configuring the item feedback

Before you can use eKomi, you have to configure the <<omni-channel/online-store/extras/customer-feedback#, customer feedback>> menu. Now insert template variables and code into the template *ItemViewSingleItem*. Choose the setting *5* (stars) as the *maximum rating points*.
