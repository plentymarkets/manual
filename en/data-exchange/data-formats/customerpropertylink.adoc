= CustomerPropertyLink
:lang: en
include::{includedir}/_header.adoc[]
:keywords: Data format CustomerPropertyLink, customer properties and values
:description: Use the data format CustomerPropertyLink to export, edit and import customer properties and the values saved for your customers.
:position: 77

The data format **CustomerPropertyLink** is used to export, edit and import <<crm/managing-customer-data#400., **customer properties** >> and values saved in the **Characteristics** tab of your customer data. Use this data format to quickly link customer properties with certain customers. Create and export the data format in the **Data exchange » Export » Dynamic export** menu. Import the data format in the **Data exchange » Import » Dynamic import** menu. The **<<data-exchange/exporting-data#, Exporting data  >>** and **<<data-exchange/importing-data#, Importing data  >>** pages of the manual describe how you export and import the data format.

Pay close attention to the exact spelling of the data fields. The data cannot be processed if the data fields are spelled incorrectly.

toc::[]

== Overview of the data fields

Table 1 explains the data fields for the **CustomerPropertyLink** data format in alphabetical order as well as filter options that you set in the **Filter** tab of the data format. The data format may vary depending on your plentymarkets version.

.Data fields of the data format **CustomerPropertyLink**
[cols="a,a,a"]
|====
|Field name |Explanation |Filter

|**CustomerID**
|The ID of the customer
|

|**CustomerPropertyID**
|The ID of the customer property
|Activate the filter, select the operator and enter the value.

|**CustomerPropertyValue**
|The value saved for the customer within the customer property.
|Enter a value to link a property with a customer. +
**__Note__**: If you leave the field **CustomerPropertyValue** blank or do not activate it, then the property will be deactivated for the customer. +
Go to **Settings » Customers » Characteristics** to create properties.

|**Export for customer class**
|**__Note__**: filter setting only
|Activate the filter and select the customer class from the drop-down menu. In order to export data for all customer classes, do not activate the filter. Go to <<crm/managing-customer-data#100., **Settings » Customers » Customer classes** >> to create customer classes.
|====

== Overview of the mandatory fields

The following mandatory fields must be available for the data format **CustomerPropertyLink** and need to have a valid value saved for them:

* **CustomerPropertyID**
* **CustomerID**
