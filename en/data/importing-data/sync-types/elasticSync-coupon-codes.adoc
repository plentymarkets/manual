With a sync of the type *Coupon code*, you import coupon codes.

[#100]
== Fields for matching coupon code data

The following fields are available for matching coupon code data. During the import, plentymarkets checks if the imported data record already exists based on the matching field.

* Campaign ID

[#200]
== Fields for mapping coupon code data

<<table-mapping-fields-coupon-codes>> explains the fields that are available for matching coupon code data. During the import, plentymarkets checks if the imported data record already exists based on the matching field.

[[table-mapping-fields-coupon-codes]]
.Mapping fields for importing coupon code data
[cols="1,1,2"]
|====
|Mapping field |Permissible import value |Explanation

|Campaign ID
|Numeric (internal ID)
|ID of the campaign

|Coupon code
|Alpha-numeric
|Coupon code

|Discount value
|Numeric
|Discount value of the coupon

|Number of redemptions
|Numeric
|Number how often the coupon has already been redeemed.

|Amount redeemed
|Numeric
|Amount of the coupon that has already been redeemed. +
*_Important:_* During import, not the residual balance of the coupon is imported, but the amount already redeemed. +
*_Example:_* The coupon has a value of EUR 100.00. The customer redeems EUR 80.00 of it. Thus, the residual balance is EUR 20.00. The value that is imported with this mapping field, is EUR 80.00.

|====
