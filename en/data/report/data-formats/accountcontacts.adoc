= accountContacts
:lang: en
include::{includedir}/_header.adoc[]
:keywords: accountContacts, report, raw data
:position: 10

The data format creator *accountContacts* exports the contacts (customer data) available in your plentymarkets system and saves the data on a cloud storage.

== Technical specifications

<<table-specifications-accountcontacts>> explains the available properties and the corresponding values.

[[table-specifications-accountcontacts]]
.Technical specifications of the *accountContacts* format
[cols=4]
|====
|Property |Meaning |Value |Explanation

|dataName
|Name
|accountContacts
|Name of the creator process and prefix of the created file. +
File name: *accountContacts-xxx.csv*

|fileFormat
|File format
|CSV, GZIP
|The data is provided as a GZIP-compressed CSV file (RFC 1952).

|defaultActive
|Activated by default
|no
|The data is only exported automatically in the background after you have activated the creator in the *Data » Raw data* menu.

|interval
|Interval
|daily
|With this setting, all data that has been changed since the last run is exported once a day, usually during the night.

|limitPage
|Lines per request
|up to 3,000
|The maximum number of data rows that is written into a single file.

|limitDay
|Lines per day
|up to 80,000
|The maximum number of data rows that is written on one day in total.

|earliestPossibleDataDate
|Earliest point in time
|01/01/2017
|Point in time as of which the raw data can be collected. The data is filtered by the date saved in the *dataUpdatedAtColumnName* column.


|dataUpdatedAtColumnName
|Column header
|ac_updated_at
|Name of the column that contains the time stamp of the last data update.

|deleteRawDataAfterDaysMinimum
|Minimum number of days until the data is deleted
|20
|The minimum number of days until the generated raw data file is deleted from the cloud storage. Therefore, the data must be processed before this date. The exact duration can be defined in the *Data » Raw data* menu.

|deleteRawDataAfterDaysMaximum
|Maximum number of days until the data is deleted
|80
|The maximum number of days until the generated raw data file is deleted from the cloud storage. Therefore, the data must be processed before this date. The exact duration can be defined in the *Data » Raw data* menu.

|====

== Data columns

<<table-data-columns-accountcontacts>> explains the data columns of the *accountContacts* format that are saved in the CSV file during the raw data export.

[[table-data-columns-accountcontacts]]
.Data columns of the *accountContacts* format
[cols="1,3,3"]
|====
|Column name |Full name |Explanation

|plenty_id
|plenty id
|ID of the plentymarkets system

|ac_id
|account contact id
|ID of the contact

|a_id
|account id
|ID of the company, if available

|act_id
|account contact type id
|ID of the contact type +
Possible values: +
1 = customer +
2 = interested party +
3 = sales representative +
4 = supplier +
5 = manufacturer +
6 = partner

|acc_id
|account contact class id
|ID of the contact class

|r_id
|referrer id
|ID of the order referrer via which the data record was saved

|u_id
|user id
|ID of the user that was saved as owner of the contact

|ac_name
|account contact name
|First name and last name of the contact

|ac_country
|account contact country
|Country of the contact as ISO2 country code

|ac_lang
|account contact lang
|Language of the contact that is saved in the contact data record. +

|ac_rating
|account contact rating
|Rating of the contact +
Is used as internal rating of the contact. 5 grey stars mean that no rating was saved. 5 yellow stars are the best rating, 5 red stars are the worst rating: +
0 = 5 grey stars (no rating) +
5 = 5 yellow stars (best possible rating) +
4 = 4 yellow stars +
3 = 3 yellow stars +
2 = 2 yellow stars +
1 = 1 yellow star +
-1 = 1 red star +
-2 = 2 red stars +
-3 = 3 red stars +
-4 = 4 red stars +
-5 = 5 red stars (worst possible rating)

|ac_updated_at
|account contact updated at
|Time stamp of the last update +
Data format: MM/DD/YYYY HH:MI:SS TZ

|ac_created_at
|account contact created at
|Time stamp of the creation +
Data format: MM/DD/YYYY HH:MI:SS TZ

|====
