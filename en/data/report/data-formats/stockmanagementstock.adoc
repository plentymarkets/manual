include::{includedir}/_header.adoc[]

The data format creator *stockManagementStock* exports the stocks available in your plentymarkets system and saves the data on a cloud storage.

== Technical specifications

<<table-specifications-stockmanagementstock>> explains the available properties and the corresponding values.

[[table-specifications-stockmanagementstock]]
.Technical specifications of the format *stockManagementStock*
[cols=4]
|====
|Property |Meaning |Value |Explanation

|dataName
|Name
|stockManagementStock
|Name of the creator process and prefix of the created file. +
File name: *stockManagementStock-xxx.csv.gz*

|fileFormat
|File format
|CSV, GZIP
|The data is provided as a GZIP-compressed CSV file (RFC 1952).

|limitPage
|Lines per request
|up to 10.000
|The maximum number of data rows that is written into a single file.

|limitDay
|Lines per day
|up to 200.000
|The maximum number of data rows that is written on one day in total.

|earliestPossibleDataDate
|Earliest point in time
|01-Jan-2006
|Point in time as of which the raw data can be collected. The data is filtered by the date saved in the *dataUpdatedAtColumnName* column.

|deleteRawDataAfterDaysMinimum
|Minimum number of days until the data is deleted
|20
|The minimum number of days until the generated raw data file is deleted from the cloud storage. Therefore, the data must be processed before this date. The exact duration can be defined in the *Data » Raw data* menu.

|deleteRawDataAfterDaysMaximum
|Maximum number of days until the data is deleted
|80
|The maximum number of days until the generated raw data file is deleted from the cloud storage. Therefore, the data must be processed before this date. The exact duration can be defined in the *Data » Raw data* menu.

|defaultActive
|Activated by default
|false
|The data is only exported automatically in the background after you have activated the creator in the *Data » Raw data* menu.

|interval
|Interval
|daily
|With this setting, all data that has been changed since the last run is exported once a day, usually during the night.

|====

== Data columns

<<table-specifications-stockmanagementstock>> explains the available properties and the corresponding values.

[[table-specifications-stockmanagementstock]]
.Data columns of the format *stockManagementStock*
[cols="1,3,3"]
|====
|Column name |Full name |Explanation

|plenty_id
|plenty id
|ID of the plentymarkets system

|iv_id
|item variation id
|ID of the item variation

|smw_id
|stock management warehouse ID
|ID of the warehouse

|sms_physical
|stock management stock physical
|Quantity of items that are physically available in the warehouse (physical stock)

|sms_reserved
|stock management stock reserved
|Quantity of stocked items that are reserved (partial quantity of the physical stock)

|sms_net
|stock management stock net
|Quantity of stocked items available for sale (physical stock minus reserved stock = net stock)

|sms_reordered
|stock management stock reordered
|Quantity of reordered items for which incoming items has not yet been booked

|sms_average_purchase_price
|stock management stock average purchase price
|Average purchase price of the item, depending on the <<stock-management/new-incoming-items#250, settings>> made beforehand.

|sms_updated_at
|stock management stock updated at
|Time stamp of the last update +
Date format: MM/DD/YYYY HH:MI:SS TZ

|====
