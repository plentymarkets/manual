The elastic export is used to export data from your plentymarkets system. There are two possible ways to use the elastic export:

* Export data from your plentymarkets system. To do so, first use the link:https://marketplace.plentymarkets.com/en/plugins/integration/formatdesigner_6483[FormatDesigner^] to create so-called data formats. In other words, you'll decide which data should be exported. Second, use the link:https://marketplace.plentymarkets.com/en/plugins/channels/marktplaetze/elasticexport_4763[Elastic Export^]{nbsp}icon:external-link[] to export the data formats that you have just created.

* Create export formats for price search engines, price comparison portals, markets, and affiliate service providers in order to export data to the interfaces. Further information can be found in the <<Creating export formats for interfaces, Creating export formats for interfaces>> chapter and in the user guides for the interface plugins.

[TIP]
.Install the Elastic Export plugin
====
You have to install the link:https://marketplace.plentymarkets.com/en/plugins/channels/marktplaetze/elasticexport_4763[Elastic Export^] plugin in your plentymarkets system first in order to be able to use the elastic export. The plugin is available in the link:https://marketplace.plentymarkets.com/en[plentyMarketplace^]{nbsp}icon:external-link[].
====

[#50]
== Creating export formats for interfaces

Use the *Data » Elastic export* menu to create export formats for price search engines, price comparison portals, markets and affiliate service providers in order to export data to the interface partners. In order to use the elastic export, install the link:https://marketplace.plentymarkets.com/en/plugins/channels/marktplaetze/elasticexport_4763[Elastic export^] plugin in your plentymarkets system. The *Elastic export* plugin is available in the link:https://marketplace.plentymarkets.com/en[plentyMarketplace^]{nbsp}icon:external-link[]. +

Further export formats for the elastic export are available in the plentyMarketplace. In order to use an export format for the elastic export, install a plugin in your plentymarkets system.

[TIP]
.Plugin set linked to default client
====
Install the link:https://marketplace.plentymarkets.com/en/plugins/channels/marktplaetze/elasticexport_4763[Elastic Export^]{nbsp}icon:external-link[] plugin and the plugins for the export formats in a plugin set that is linked to the default client.
====

[#300]
=== Creating export formats

This chapter teaches you how to create new export formats and which settings you can carry out. All data formats, except *BolOfferExport *and *BolItemExport*, have to be installed from the link:https://marketplace.plentymarkets.com/en[plentyMarketplace^]{nbsp}icon:external-link[] first.

[.instruction]
Creating a new export format:

. Go to *Data » Elastic export*.
. Click on *New export*.
. Carry out the settings. Pay attention to the explanations given in <<settings-elastic-export>>.
. *Save* (icon:save[role="green"]) the settings. +
→ The export format is given an ID and it appears in the overview within the *Exports* tab.

[NOTE]
.Item filters for the elastic export
====
Please note that no item filters for the elastic export are set by default. If you do not add any item filters, all items are exported.
====

2+| Settings

|*Name*
| Enter a name. The export format is listed by this name in the overview within the *Exports* tab.

|*Type*

|*Generate cache file*
| Place a check mark if you want to transfer more than 9,999 data records to the interface. We recommend that you do not activate this setting for more than 20 export formats. This is to ensure a high performance of the elastic export.

|*Provisioning*
|Select *URL*, *FTP* or *Admin area*. +
Different settings will appear depending on which option is selected here. +
*URL*: Use this option to generate a token for authentication in order to allow external access. +
*FTP*: Server on which the file is saved. +
*Admin area*: External access is not possible.

|*Token*, +
*URL*

*Port*, +
*User name*, +
*Password*, +
*Path*, +
*Transfer*
| If you selected the option *FTP* under *Provisioning*, then enter the parameters. The standard FTP port is *21*. +

|*File name*
| If you selected the option *Admin area* under *Provisioning*, then enter the file name. Each provisioning type requires that the file name has the ending of the file format, e.g. use the suffix .csv for a CSV file.

2+| Item filters

|*Add item filters*
a|Select the items you wish to export. To do so, select an item filter from the drop-down list and click on *Add* (icon:plus-square[role="green"]). There are no filters set by default. It is possible to add multiple item filters from the drop-down list one after the other.

* *Active* = Only active variations are exported.
* *Item type* = Only items of the selected item type are transferred.
* *Stock* = Select which stocks should be exported.
* *Image* = Activate to transfer the item with its image. Only items with images are transferred.
* *Manufacturers* = Select one, several or *ALL* manufacturers.
* *Category* = Activate to transfer the item with its category link. Only items belonging to this category are exported.
* *Client* = Select a client.
* *Flag 1-2* = Select a flag.
* *Markets* = Select one market, several or *ALL*. The availability for all markets selected here has to be saved for the item. Otherwise, the export will not take place.
* *Positive stock for given warehouse* = Select a warehouse from the drop-down list that opens to only export variations with positive stock in this warehouse. +
  *_Note:_* This stock filter only applies to FormatDesigner formats of the type *Item*.
* *Prices* = Only items with the selected price are exported. +
  ** *Without price* = Items without price are transferred. +
  ** *With price* = Items with price are transferred. +
  ** *With price lower* = Enter a price. Items with a lower price than the price that was entered are exported. +
  ** *With price higher* = Enter a price. Items with a higher price than the price that was entered are exported. +
  ** *With price between* = Enter a lower limit and upper limit. Items with a price between the two values are exported.
* *Tag (AND mode)* = Enter one or more tag names. Items that are linked to _all_ of the selected <<welcome/general-functions/tags#, tags>> are exported.
* *Tag (OR mode)* = Enter one or more tag names. Items that are linked to _at least one_ of the selected <<welcome/general-functions/tags#, tags>> are exported.
* *Variations* = Select the variations to be transferred. +
  ** *Transfer all* = All variations are transferred. +
  ** *Only transfer main variations* = Only main variations are transferred. +
  ** *Do not transfer main variations* = Only the other variations of an item are transferred. Main variations are not transferred. +
  ** *_Tip:_* Use this option if the main variations are virtual and not for sale. +
  ** *Only transfer single variations* = Only those main variations of items are transferred that only have a main variation and no other variations.
* *Currency* = Select a currency.

|*Language*

|*Item name*

Enter a number into the *Maximum number of characters (def. text)* field if desired. This defines how many characters are exported for the item name.

|*Preview text*

Enter a number into the *Maximum number of characters (def. text)* field if desired. This specifies how many characters are exported for the preview text. +
Activate the *Remove HTML tags* option if you want HTML tags to be removed during the export. +

|*Description*

If you only want to allow specific HTML tags to be exported, then enter these tags into the *Permitted HTML tags, separated by comma (def. text)* field. Use commas to separate multiple tags.

=== Editing export formats

All of the export formats that you created are displayed in the overview in the *Data » Elastic export » Tab: Exports* menu. Use the *Customise the layout* function to select the columns to be displayed. Point your cursor over the toolbar and click on *Customise the layout* (icon:cog[]). The *Customise the layout* window opens and you can activate or deactivate the columns. The following columns are available:

* ID
* Name
* Download
* Type
* Format
* Limit
* Last export change
* Export created at
* Delete

Open an export format to check or edit its settings.

[.instruction]
Editing an export format:

. Go to *Data » Elastic export*.
. Open the export format.
. Carry out the desired changes. Pay attention to the explanations given in <<settings-elastic-export>> and <<toolbar-export-format>>.
. *Save* (icon:save[role="green"]) the settings.

=== Filtering export formats

You can use specific criteria to filter your export formats. This helps to reduce the results displayed in the *Exports* tab.

[.instruction]
Filtering export formats:

. Go to *Data » Elastic export*.
. Carry out the settings in the *Filter* section. Pay attention to the explanations given in <<filter-settings-elastic-export>>.
. Click on *Search* (icon:search[role="blue"]).

[[filter-settings-elastic-export]]
.Filter settings in the *Data » Elastic *export menu
[cols="1,3"]
|====
| Filter | Explanation

|*ID*
| Enter the ID to search for the variation with this ID.

|*Name*
| Enter part of a name or the full name.

|*Type*
| Select the type *item* or *All* from the drop-down list.

|*Format*
| Select a format from the drop-down list. A list of all the export formats that match the selected format is displayed.
|====

[NOTE]
.Deleting search criteria
====
Click on *Reset* (icon:undo[role="orange"]) to delete the search criteria. Then click on *Search* (icon:search[role="blue"]) again. All of your export formats are displayed in the overview and you can select new filters.
====
