== Setting up Zalando in plentymarkets

As soon as Zalando has accepted you into their partner programme, you will receive a user guide from Zalando which helps you setting up the interface. In plentymarkets, carry out the following basic settings.

[.instruction]
Setting up Zalando in plentymarkets:

. Go to *Setup » Markets » Zalando » Settings*. +
→ The *Settings* tab opens.
. Carry out the desired settings. Pay attention to the explanations given in <<basic-settings-zalando>>.
. *Save* (icon:save[role="green"]) the settings.

[cols="1,3"]

2+|Basic configuration

|*User*
|Select the API user. The account for the API user is created during the initial setup.

|*Price column*
|Select a price column that is used for Zalando items.

|*Item name*
|Select *Name 1*, *Name 2* or *Name 3*.

|*Assignment client (store)*

== Activating the order referrer

In order to link items, properties etc. with Zalando, you have to activate the order referrer Zalando in the *Setup » Orders » Order referrer* menu.

[.instruction]
Activating the order referrer for Zalando:

. Go to *Setup » Orders » Order referrer*.
. Place a check mark next to *Zalando*.
. *Save* (icon:save[role="green"]) the settings.

[#300]
== Setting the item availability

Items have to be available for Zalando. This is done in the *Availability* tab of an item within the *Item » Edit item » Open item » Tab: Variation ID* menu.

[.instruction]
Setting the item availability for Zalando:

. Go to *Item » Edit item » Open item » Tab: Variation ID » Tab: Settings*.
. Activate the main variation in the *Availability* section.
. Click on the *Availability* tab.
. Click in the selection field in the *Markets* section.  +
→ A list with all available markets is displayed.
. Place a check mark next to the option *Zalando*.
. Place a check mark next to the option *Web API*.
. Click on *Add* (icon:plus-square[role="green"]). +
→ The market is added.
. *Save* (icon:save[role="green"]) the settings. +
→ The item is available on Zalando.

The availability for variations can be edited in the *Item » Edit item » Open item » Tab: Variations » Open variation » Tab: Variation ID » Tab: Availability* menu.

== Defining SKU manually

Add SKUs manually for specific referrers in the *Item » Edit item » Open item » Tab: Variation ID » Tab: Availability* menu if needed.

[.instruction]
Adding SKU:

. Go to *Item » Edit item » Open item » Tab: Variation ID » Tab: Settings*.
. Click on the *Availability* tab.
. Click on *Add* (icon:plus-square[role="green"]) in the *SKU* section. +
→ The *New SKU* window is displayed.
. Select the referrer *Zalando*.
. Enter the SKU.
. Click on *Add* (icon:plus-square[role="green"]). +
→ The SKU is saved and displayed.

== Linking categories

Link your online store categories to the Zalando categories in order to display your items in these categories. Further items from the linked category are assigned automatically.

[.instruction]
Linking categories:

. Go to *Setup » Markets » Zalando » Category link*.
. Select a category under *Zalando category*.
. *Save* (icon:save[role="green"]) the settings.

[#500]
== Converting silhouettes to properties

In the *Setup » Markets » Zalando » Settings » Tab: Silhouettes* menu, you convert Zalando silhouettes to item properties. Silhouettes are transferred when Zalando is first set up.

. Go to *Setup » Markets » Zalando » Settings*.
. Click on the *Silhouettes* tab.
. Select the *Groups* for the properties that you want to convert.
. *Save* (icon:save[role="green"]) the settings. +
→ The properties are created in your plentymarkets system.

[#600]
== Exporting Zalando item properties

You can export the silhouettes that you saved in the *Groups* section and use them for the dynamic data import.

[.instruction]
Exporting Zalando item properties:

. Go to *Setup » Markets » Zalando » Settings*.
. Click on the *Silhouettes* tab.
. Click on *Export Zalando item properties*. +
→ A file containing the property parameters is exported.
. Open the file.
. Carry out the settings. Pay attention to the explanations given in <<zalando-properties>>.
. *Save* (icon:save[role="green"]) the settings.

== Linking attributes

If you have created attributes in your plentymarkets system, link these attributes with Zalando. The following example describes how you can link the attribute “colour” with Zalando. Link further attributes in the same way.

[.instruction]
Linking the attribute “colour”:

. Go to *Setup » Item » Attributes*. +
→ An overview of all existing attributes opens.
. Open the attribute *colour*.
. Click on the *Market variation* tab.
. Select the option *Zalando* from the *Market* drop-down list. +
→ The *Zalando* drop-down list is displayed.
. Select the attribute *colour* from the *Zalando* drop-down list.
. *Save* (icon:save[role="green"]) the settings.

== Order status changes

When an order status changes, this information is not transferred to Zalando with event procedures, but retrieved automatically via an interface. The following statuses are transferred to Zalando: shipping confirmation, cancellation, and return.

The order statuses shipping confirmation, cancellation, and return are sent to Zalando if the following criteria are fulfilled:

* *Shipping confirmation*: The outgoing items have been booked. In addition, a return label was created and package numbers were added.

* *Cancellation*: The order moved to *status [8.X]* or the quantity of an order item is *0*.

* *Return*: A return was created in plentymarkets.

== Retrieving processing reports

In the *Setup » Markets » Zalando » Settings » Tab: processing reports* menu, you can retrieve processing reports as well as the export log for Zalando.

[.instruction]
Retrieving processing reports:

. Go to *Setup » Markets » Zalando » Settings*.
. Click on the *Processing reports* tab.
. Click on *Download file* under *Procedures* in the line of a processing report. +
→ A CSV file is downloaded.

Download the export log in the *Export log* section.

== Viewing the API log

The *Data » API log* menu contains a history of processes that run via the Zalando interface. The *ZalandoController* format is available for Zalando.

[.instruction]
Viewing the API log:

. Go to *Data » API log » Tab: API log*.
. Select the *ZalandoController* format from the *Process* drop-down list.
. Select a *date* if you only want to see entries for a specific day.
. Select the number of results per page.
. Click on *Search* (icon:search[role="blue"]) to display the results.
