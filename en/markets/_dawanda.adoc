link:http://de.dawanda.com[DaWanda^] is an online marketplace for unique, handmade, individualised and custom-made products and low volume productions. You are not allowed to sell mass-produced or imported items on DaWanda. There are special categories such as Material and Vintage where you can offer industrially manufactured products. These products have to comply with the link:https://en.dawanda.com/page/policies[DaWanda Guidelines^]. In order to sell items on DaWanda, you need to make your items completely available in German.

[#100]
== Registering with DaWanda

You need to register with link:https://de.dawanda.com/account/register[DaWanda^] and have your product line reviewed before setting up DaWanda in plentymarkets. If you already sell on DaWanda, you need to request your API key. To do so, contact DaWanda: link:mailto:feedback@dawanda.com[feedback@dawanda.com]. Make sure to include the email address that you saved for DaWanda.

== Setting up DaWanda in plentymarkets

Set up DaWanda in plentymarkets to offer items on DaWanda. To do so, proceed as follows.

[.instruction]
Setting up DaWanda in plentymarkets:

. Go to *System » Markets » DaWanda » Settings*.
. Carry out the settings. Pay attention to the information given in table 1.
. *Save* the settings.

.Basic settings for the market DaWanda
[cols="1,3"]
|====
|Setting |Explanation

|*Activate automatic data exchange*
|If the data exchange is activated, then there is a daily item export, an hourly stock synchronisation and an hourly order import.

|*Activate item export*
|If this option is activated, a daily item export is carried out.

|*Activate stock update*
|If this option is activated, an hourly stock update is carried out.

|*Deactivate the import of the E-Mail addresses*
|Activate this option if you do not want to import the email addresses from the customer data.

|*API key*
|Enter the API key that you received from DaWanda.  +
The API key is checked after the data has been saved. The fields *Shop ID* and *Shop name* are automatically filled in after the API key has been checked.

|*Shop ID*
|The DaWanda store ID. This information is automatically filled in after the API key has been checked.

|*Shop name*
|The name of the DaWanda store. This information is automatically filled in after the API key has been checked.

|*Item name*
|Select *Name 1*, *Name 2* or *Name 3*. This item name is transferred to DaWanda.

|*Short item description*
|Select *Preview text*, *Name 1*, *Name 2* or *Name 3*. This short item description is transferred to DaWanda.

|*Item description*
|Select the *Item text* saved in the item's *Texts* tab or select one of the free text fields saved in the item's *Fields* tab. This item description is transferred to DaWanda.

|*Import only confirmed orders*
|If this setting is activated, then only orders which have been confirmed on DaWanda are imported.

|*Variation ID for the additional charge*
|Enter a variation ID that should be assigned to the additional costs. This variation is assigned to the additional costs during the order import. +
We recommend to use a variation ID for the additional charge if you have activated the automatic stock update. Otherwise, paid orders will be set to status *[4] In preparation for shipping*. This might lead to a delay in the shipping process.
|====

[IMPORTANT]
.Variations are transferred to DaWanda as individual items
====
It is not possible to set up variations in DaWanda. Therefore, an individual item is created for every variation on DaWanda. Colour and size are automatically added to the title. +
Tip: Item names can have a maximum of 50 characters on DaWanda. Longer item names will be cut off. If an item name is too long with information about colour and size, then use *Name 1*, *Name 2* or *Name 3* in the item's *Texts* tab and transfer these item names to DaWanda.
====

Connecting to this market allows for the following automatic processes to take place:

* daily item export
* hourly stock synchronisation
* hourly order import

[TIP]
.Item image
====
A maximum of 4 item images can be uploaded to DaWanda.
====

In order to link items, properties etc. with DaWanda you have to activate the order referrer DaWanda in the menu *System » Orders » Order referrer*.

Activating the order referrer for DaWanda:

. Go to *System » Orders » Order referrer*.
. Place a check mark next to *DaWanda*.
. *Save* the settings.

== Setting the item availability

Items have to be available for DaWanda. This is done in the *Availability* tab of an item within the <<en/item/managing-items#170, Item » Edit item » Open item » Tab: Variation ID>> menu.

[.instruction]
Setting the item availability for DaWanda:

. Go to *Item » Edit item » Open item » Tab: Variation ID » Tab: Settings*.
. Activate the main variation in the *Availability* section.
. Click on the *Availability* tab.
. Click in the selection field in the *Markets* section.  +
→ A list with all available markets is displayed.
. Activate the option *DaWanda*.
. Activate the option Web API.
. Click on *Add*. +
→ The market is added.
. *Save* the settings. +
→ The item is available on DaWanda.

The availability for variations can be individually edited in the *Item » Edit item » Open item » Tab: Variations » Open variation » Tab: Variation ID » Tab: Availability* menu.

→ The window *New SKU* is displayed.

→ The SKU is saved and displayed.

== Linking categories

Link your store categories to the DaWanda categories in order to display your items in these DaWanda categories. Further items from the linked category are assigned automatically.

[.instruction]
Linking categories:

. Go to *System » Markets » DaWanda » Category link*.
. Click on *Search*.  +
→ The window *Select category* opens.
. Select the DaWanda category that best matches your store category.
. Click on *Accept*. +
→ The ID and category path of the DaWanda category are displayed.
. If you already know the ID of the DaWanda category, enter it directly into the *Market category* field to link it to your store category.
. *Save* the settings.

== Creating properties

Use <<n/item/managing-items#480, properties>> to link your items to particular characteristics that you need for the market DaWanda. This way, you can link categories directly in the item menu. Moreover, you transfer item information to DaWanda with the help of property filters and determine criteria for the item search on DaWanda.

First, create properties and then link them to items. To do so, proceed as follows.

[.instruction]
Creating a property:

. Go to *System » Item » Properties*.
. Click on the *New property* tab.
. Enter a name for the property under *Name (internal)*.
. Select a *Property type*.
. Select a *DaWanda property* from the drop-down list. Pay attention to the explanations given in table 3.
. *Save* the settings.

.DaWanda properties
[cols="1,3"]
|====
|Property |Explanation

|*Type of product*
|Select between *physical product*, *digital product* or *service*. +
Property type: No

|*AutoRenewEnabled*
|Activate or deactivate the automatic renewal of offers.  +
Go to *Item » Edit item » Open item » Tab: Properties* and enter *true*, *yes*, *ja*, *1* for the DaWanda property or link a property of type *No* = The offer is automatically renewed after the runtime ends.  +
*Note:* *False* is automatically transferred for items that do not use the *AutoRenewEnabled* property or have invalid values. The offer is not renewed automatically. +
Property type: No

|*Own category ID*
|It is possible to create your own DaWanda store categories which are linked to items in plentymarkets by using properties. Enter the DaWanda store category ID here. Request the IDs directly from DaWanda by download. +
Property type: whole number

|*Colour1*
|The item’s main colour 1. This is also used for the item search on DaWanda. +
Property type: No

|*Colour2*
|The item's main colour 2. This is also used for the item search on DaWanda. +
Property type: No

|*Size/Dimensions/Weight*
|Enter detailed information about the item's size, dimensions or weight. +
Property type: Whole number

|*Production method*
|Enter information about the production method, e.g. hand-made. +
Property type: Text

|*Customisation options*
|Describe the ways in which the item is customisable. For example patterns, sizes or colours that differ from the item displayed. Also specify how the customer can communicate his or her customisation wishes and whether the price will change due to customisation. +
Property type: Text

|*Category ID*
|Enter a category ID that differs from the categories linked in the *System » Markets » DaWanda » Category link* menu. +
Property type: Whole number

|*Keywords*
|Enter additional information. +
Property type: Text

|*Shipping profile*
|Select a shipping profile. Only visible if shipping profiles have already been entered on DaWanda. Due to the cache function, the property is not displayed with the item until about 1 hour after it was saved. +
*Important:* The property *Shipping profile* is mandatory for DaWanda. +
Property type: No

|*Materials utilised*
|Enter all of the materials that were used for the item and additional care instructions if needed. +
Property type: Text

|*Essential properties*
|Enter the *link:http://de.dawanda.com/cms/c/de/Verkauferportal/22-News/78-Verkaufernewsletter/p2337-News-fur-Verkaufer-Gesetzesanderung-seit-dem-1-August-2012[essential properties^]* of the item according to the requirements of the so-called button solution in Germany. The main aim of these regulations is to protect consumers from unclear contracts (relating to subscriptions and costs). +
Property type: Text

|*Cancellation rights*
|Select the cancellation rights. Only visible if cancellation rights have already been entered on DaWanda. Due to the cache function, the property is not displayed with the item until about 1 hour after it was saved. +
Property type: No

|*Second category ID*
|Enter the second category ID that differs from the categories linked in the *System » Markets » DaWanda » Category link* menu. +
*Note:* DaWanda charges an additional fee for listing items in a second category. +
Property type: Whole number
|====

== Activating payment methods

Activate the <<payment/managing-payment-methods#, payment method>> *DaWanda Gutschein* if you want to allow customers to pay with coupons from DaWanda. Then activate the payment method *DaWanda Payment* in the same way.

[.instruction]
Activating DaWanda payment methods:

. Go to *System » Orders » Payment » Methods*.
. Click on the folder *International* > *DaWanda*.
. Select the DaWanda payment method *DaWanda Gutschein*. +
→ The settings for the DaWanda payment method are displayed.
. Carry out the settings. Pay attention to the information given in table 2.
. Place a check mark next to the option *Active*.
. *Save* the settings.

.Settings for the payment method
[cols="1,3"]
|====
|Setting |Explanation

|*Language*
|The following options are saved for the language selected: *Name*, *Info page*, *Info page (internal)*, *Logo* and *Upload logo*. This makes it possible to enter values for these options in various languages. The values for a language are activated as soon as a buyer selects the language in the online store.

|*Name*
|Enter the name that should be displayed in the online store and on invoices etc.

|*Info page*
|Select a category page of the type *content* or enter the URL of a website to provide information about the payment method.

|*Info page (external)*
|If you have selected the option *Info page (external)* under *Info page*, then enter the URL of the info page here.

|*Countries of delivery*
|Do not select a country of delivery. +
You cannot use the payment method in your online store. Therefore, this payment method is only activated in plentymarkets. It is not completely set up.

|*Logo*
|Select one of the following options for displaying the logo: +
*Show standard logo* = The logo that is saved in plentymarkets for this payment method is displayed. +
*Do not show a logo* = The logo that is saved in plentymarkets for this payment method is not displayed. +
*Show upload logo* = The options *Logo view* and *Upload logo* are displayed.

|*Logo view*
|If *Show upload logo* has been selected from the *Logo* drop-down list, then the logo that has been uploaded is displayed here.

|*Upload logo*
|If *Show upload logo* has been selected from the *Logo* drop-down list, upload an external logo here. +
 Click on *Upload files* and select the logo that you want to upload for the payment method. Permitted data formats: GIF, PNG and JPG.

|*Priority*
|Select the priority for displaying the payment method in the online store.

|*Costs: Flat rate or percentage*
|If the payment method results in additional costs, enter the percentage value or flat rate value. The choice depends on the conditions of your contract. *Important*: Do not enter a value into both fields.  +
For further information, refer to the <<en/basics/statistics, Statistics>> page of the manual. For an example of flat rate and percentage costs, refer to the <<payment/managing-payment-methods#20, Managing payment methods>> page of the manual.
|====

== Viewing the API log

The menu <<en/basics/data-exchange/api-log#, Data » API log>> contains a history of processes that run via the DaWanda interface.

[.instruction]
Viewing the API log:

.  Go to *Data » API log » Tab: API log*.
.  Select the format *DaWanda.de* from the *Process* drop-down list.
.  Select a date if you only want to see entries for a specific day.
.  Click on *Search* to carry out the search. +
→ The processes are displayed.
