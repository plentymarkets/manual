= DaWanda
:lang: en
include::{includedir}/_header.adoc[]
:description: Multi-Channel in plentymarkets: Set up the interface to the market DaWanda in your plentymarkets system.
:position: 40

*link:http://en.dawanda.com/[DaWanda^]* is an online marketplace for unique, handmade, individualised and custom-made products and low volume productions. You are not allowed to sell mass-produced or imported items on DaWanda. There are special categories such as Material and Vintage where you can offer industrially manufactured products. These products have to comply with the *link:http://en.dawanda.com/cms/c/en/Seller-Portal/537-Guidelines[DaWanda Guidelines^]*. In order to sell items on DaWanda, you need to make your items completely available in German.

[#100]
== Registering with DaWanda

You need to register with *link:https://de.dawanda.com/account/register[DaWanda^]* and have your product line reviewed before setting up DaWanda in plentymarkets. If you are already selling on DaWanda, you need to request your API key. To do so, contact DaWanda: link:mailto:feedback@dawanda.com[feedback@dawanda.com]. Make sure to include the email address that you saved for DaWanda.

== Setting up DaWanda in plentymarkets

Set up DaWanda in plentymarkets to offer items on DaWanda. To do so, proceed as follows.

[.instruction]
Setting up DaWanda in plentymarkets:

. Go to *System » Markets » DaWanda » Settings*.
. Carry out the settings as desired. Pay attention to the information given in table 1.
. *Save* the settings.

.basic settings for the market DaWanda
[cols="a,a"]
|====
|Setting |Explanation

|*Activate the automatic data exchange*
|If the data exchange is activated, then there will be a daily item export, an hourly stock synchronisation and an hourly order import.

|*API key*
|Enter the API key that you received from DaWanda.  +
The API key will be checked after the data is saved. The fields *Shop ID* and *Shop name* will automatically be filled in after the API key is checked.

|*Shop ID*
|The DaWanda store ID. This information is automatically filled in after the API key is checked.

|*Store name*
|The name of the DaWanda store. This information is automatically filled in after the API key is checked.

|*Item name*
|Select *Name 1*, *Name 2* or *Name 3*. This item name will be transferred to DaWanda.

|*Short item description*
|Select *Preview text*, *Name 1*, *Name 2* or *Name 3*. This short item description will be transferred to DaWanda.

|*Item description*
|Select the *Item text* saved in the item's *Texts* tab or select one of the free text fields saved in the item's *Fields* tab. This item description will be transferred to DaWanda.

|*Import only confirmed orders*
|Only orders that are confirmed on DaWanda will be imported.
|====

[IMPORTANT]
.Variations are transferred to DaWanda as individual items
====
It is not possible to set up variations in DaWanda. Therefore, an individual item is created for every variation in DaWanda. Colour and size are automatically added to the title. +
Tip: Item names can have a maximum of 50 characters in DaWanda. Longer item names will be cut off. If an item name is too long with information about colour and size, then use *Name 1*, *Name 2* or *Name 3* in the item's *Texts* tab and transfer these item names to DaWanda.
====

Connecting to this market allows for the following automatic processes to take place:

* daily item export
* hourly stock synchronisation
* hourly order import

[TIP]
.Item image
====
A maximum of 4 item images can be uploaded to DaWanda.
====

== Setting the item availability

Items have to be available for DaWanda. This is done in the *Availability* tab of an item within the <<item/managing-items#170, *Item » Edit item » Tab: Variation ID*>> menu.

[.instruction]
Setting the item availability for DaWanda:

. Open the *Item » Edit item » Open item » Tab: Variation ID » Tab: Settings* menu.
. Activate the main variation in the *Availability* section.
. Click on the *Availability* tab.
. Click in the *Markets* section in the selection field.  +
→ A list with all available markets will be displayed.
. Activate the option *DaWanda*.
. Activate the option *Web API*.
. Click on *Add*. +
→ The Market will be added.
. *Save* the settings. +
→ The item is available on DaWanda.

The availability for variations can be individually edit in the *Item » Edit item » Open item » Tab: Variation ID » Open Variante »Tab: Availability* menu.

== Linking categories

Link your store categories to the DaWanda categories in order to display your items in these DaWanda categories. Further items from the linked category will be assigned automatically.

[.instruction]
Linking categories:

. Go to *System » Markets » DaWanda » Category link*.
. Click on *Search*.  +
→ The *Select category* window will open.
. Select the DaWanda category that best matches your store category.
. Click on *Apply*. +
→ The ID and category path of the DaWanda category will be displayed.
. If you already know the ID of the DaWanda category, enter it directly into the *Market category* field to link it to your store category.
. *Save* the settings.

== Creating properties

Use <<item/managing-items#480, *properties*>> to link your items to particular characteristics that you need for the market DaWanda. This way, you can link categories directly in the item menu. Moreover, you transfer item information to DaWanda with the help of property filters and determine criteria for the item search on DaWanda.

First, create properties and then link them to items. To do so, proceed as follows.

[.instruction]
Creating a property:

. Go to *System » Item » Properties*.
. Click on the *New property* tab.
. Enter a name for the property under *Back end name*.
. Select a *Property type*.
. Select a *DaWanda property* from the drop-down menu. Pay attention to the explanations given in table 3.
. *Save* the settings.

== Activating payment methods

Activate the *<<order-processing/payment#, payment method>>* *DaWanda Gutschein* if you want to allow customers to pay with coupons from DaWanda. Then activate the payment method *DaWanda Payment* the same way.

[.instruction]
Activating DaWanda payment methods:

. Go to *System » Orders » Payment » Methods*.
. Click on the folder *International* &gt; *DaWanda*.
. Select the DaWanda payment method *DaWanda Gutschein*. +
→ The settings will be displayed for this DaWanda payment method.
. Carry out the settings as desired. Pay attention to the information given in table 2.
. Place a check mark next to the option *Active*.
. *Save* the settings.

.settings of the payment method
[cols="a,a"]
|====
|Setting |Explanation

|*Language*
|The following options are saved for the language selected: *Name*, *Info page*, *Info page (internal)*, *Logo* and *Upload logo*. This makes it possible to enter values for these options in various languages. The values for a language will be activated as soon as a buyer selects the language in the online store.

|*Name*
|Enter the name that should be displayed in the online store and on invoices etc.

|*Info page*
|Select a category page of the type *content* or enter the URL of a website to provide information about the payment method.

|*Info page (external)*
|If you selected the option *Info page (external)* under *Info page*, then enter the URL of the info page here.

|*Countries of delivery*
|Do not select a country of delivery. +
You cannot use the payment method in your online store. Therefore, this payment method is only activated in plentymarkets. It is not completely configured.

|*Logo*
|Select one of the following options for displaying the logo: +
*Show standard logo* = The logo that is saved in plentymarkets for this payment method will be displayed.  +
*Do not show a logo* = The logo that is saved in plentymarkets for this payment method will not be displayed. +
*Show upload logo* = The options *Logo view* and *Upload logo* will be displayed.

|*Logo view*
|If *Show upload logo* was selected from the *Logo* drop-down menu, then the logo that was uploaded will be displayed here.

|*Upload logo*
|If *Show upload logo* was selected from the *Logo* drop-down menu, then you will be able to upload an external logo here. +
Click on *Upload files* and select the logo that you want to upload for the payment method. Permitted data formats: GIF, PNG and JPG.

|*Priority*
|Select the priority for displaying the payment method in the online store.

|*Costs: Flat rate or percentage*
|If the payment method results in additional costs, enter the percentage value or flat rate value. The choice depends on the conditions of your contract. *Important:* Do not enter a value into both fields.  +
For further information, refer to the *<<basics/statistics#, Statistics>>* and *<<basics/statistics/general-statistics/item/tops-flops#, Tops/Flops>>* pages of the manual. For an example for flat rate and percentage costs, refer to the *<<omni-channel/multi-channel/dawanda#20, Basic settings>>* page of the manual.
|====

== Viewing the API log

The menu *<<basics/data-exchange/api-log#, Data » API log>>* contains a history of processes that run via the DaWanda interface.

[.instruction]
Viewing the API log:

.  Go to *Data » API log » Tab: API log*.
.  Select the *DaWanda.de* format from the *Process* drop-down menu.
.  Select a *Date* if you only want to see entries for a specific day.
.  Click on *Search* to carry out the search. +
→ The processes will be displayed.
