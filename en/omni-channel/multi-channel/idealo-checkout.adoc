On this page, you will find information about the settings you need to carry out in order to set up idealo Checkout in plentymarkets. You will learn how to set up the interface, how to export items to idealo and how to define sales prises.

[#100]
== Registering with idealo

You need to register with **link:http://info.fruugo.com/merchants/pages/start_selling.asp[idealo, window="_blank"]** before setting up idealo in plentymarkets.

[#200]
== Setting up idealo in plentymarkets

Carry out the following settings in plentymarkets in order to set up the interface to idealo.

[.instruction]
Setting up idealo:

. Go to **Settings » Markets » idealo Checkout » Basic settings**.
. Activate the automatic data exchange.
. Enter the API key as received from idealo.
. **Save** the settings.

Connecting to this market allows for the following automatic processes to take place:

* hourly order synchronisation

== Setting the item availability

Items have to be available for idealo Checkout. This is done by linking the item to the property **CheckoutApproved** and in the **Available** tab of an item in the **link::/[Item » Edit item]** menu. First create the property and link it with the item.

[.instruction]
Creating the property CheckoutApproved:

.  Go to <<omni-channel/multi-channel/idealo-checkout#2, **Settings » Item » Properties » Tab: New property** >>.
.  Enter the name (internal).
. Select the option **No** from the property type drop-down menu.
.  Select the property **CheckoutApproved** for **IdealoDK property**.
. **Save** the settings.

Next, you will have to link the special offer to the item in the **Item » Edit item » Open item » Tab: Variation ID » Tab: Properties** menu. Next, activate the item availability as described below.

[.instruction]
Setting the item availability for idealo:

. Open the **Item » Edit item » Open item » Tab: Variation ID » Tab: Settings** menu.
. Activate the main variation in the **Availability** section.
. Click on the **Availability** tab.
. Click in the **Markets** section in the selection field. +
→ A list with all available markets will be displayed.
. Activate the option **idealo**.
. Activate the option **Web API**.
. Click on **Add**. +
→ The Market will be added.
. **Save** the settings.

The availability for variations can be individually edit in the **Item » Edit item » Open item » Tab: Variation ID » Open variation » Tab: Availability** menu.

== Defining a sales price

Proceed as described below to define a sales price for the order referrer idealo. This price will be displayed on idealo.

[.instruction]
Defining a sales price for idealo:

. Open the menu **Settings » Item » Sales price » Open sales price » Tab: Settings**.
. Place a checkmark next to the referrer **idealo**.
. **Save** the settings.

Add SKUs manually for specific referrers in the menu <<item/managing-items#170, **Item » Edit item » Open item » Tab: Variation ID » Tab: Settings » Tab: Availability** >> if needed.

[.instruction]
Addding SKU

. Open the **Item » Edit item » Open item » Tab: Variation ID » Tab: Settings** menu.
.  Click on the **Availability** tab.
.  Click in the **SKU** section on **Add**. +
→ The editing window **New SKU** will be displayed.
.  Select the referrer **idealo**.
.  Enter the SKU.
.  Click on **Add**. +
→ The SKU will be saved and displayed.

== Setting up shipping company as shipping method

If the order is carried out by a forwarding company, the customer has to enter his phone number during the order process in order to arrange a delivery date. The shipping method shipping company is transferred as a property to idealo Checkout. Proceed as described below to create the property **FulfillmentType**.

[.instruction]
Create a property for the shipping company:

.  Go to <<omni-channel/multi-channel/idealo-checkout#2, **Settings » Item » Properties » Tab: New property** >>.
.  Enter the name (internal).
. Select the option **Text** from the **Property type** drop-down menu.
.  Select the property **FulfillmentType** for **IdealoDK property**.
. **Save** the settings.

Next, you will have to link the special offer to the item in the **Item » Edit item » Open item » Tab: Variation ID » Tab: Properties** menu. Next, set up the item export as described below.

It is also possible to transfer the additional services **TwoManHandlingPrice** and **DisposalPrice** to idealo Checkout. DisposalPrice can only be used in combination with the TwoManHandlingPrice. That means you have to create the property **TwoManHandlingPrice** as well as the property **DisposalPrice** and link them to the item. In both cases, select **Text** as the property type. Go to **Item » Edit item » Open item » Tab: Variation ID » Tab: Properties** and enter the surcharge for the additional service with two decimal places behind the decimal separator. Use a period for the decimal separator (19.99).

== Setting up the item export

The item export for idealo.de is carried out by the elastic export. Therefore, the plugins **ElasticExport** and **ElasticExportIdealoDE** have to be already installed. Create the data format IdealoDE first and then transfer the URL to idealo.

[.instruction]
Setting up the item export:

. Go to **Data exchange » Export » Elastic export**.
. Click on **New export**.
. Carry out the settings as desired. Pay attention to the information given in table 1.
. Carry out further settings for item filters and format settings. Pay attention to the information given in table 5 in the <<basics/data-exchange/exporting-data#60, **Exporting data** >> page of the manual.
. **Save** the settings. +
→ The export format will be given an ID and it will appear in the overview within the **Exports** tab.

.settings for the item export
[cols="a,a"]
|====
|Setting |Explanation

2+|Settings

|**Name**
|Enter a name. The export format will be listed under this name in the overview within the **Exports** tab.

|**Type**
|Select a type from the drop-down menu. Currently, **Item** is the only available option. Further options will be available soon.

|**Format**
|Select the format **IdealoDE** from the drop-down menu.

|**Limit**
|Enter a number. If you want to transfer more than 10,000 data records to idealo, then the output file will not be generated again for another 24 hours. This is to save resources.

|**Provisioning**
|Select **URL**.

|**Token, +
URL**
|If you selected the option **URL** under **Provisioning**, then click on **Generate token**. The token will be entered automatically. The URL will be entered automatically if the token was generated under **Token**. Give this URL to idealo in order to export the items.
|====

[IMPORTANT]
.Export information
====
If you change one or more settings, then you will have to transfer the new link to idealo.

A maximum of 4000 items will be exported at once. If you export more items, then a second export will automatically be carried out. To display the second export process, set the parameter **&amp;Page** = in the export link to **2**.

Technical data is transferred to idealo in the last column of the export file.
====

== Viewing the API log

The menu **Data exchange » API log » Tab: API log** contains a history of processes that run via the idealo interface. The **idealoDirektkauf** format is available for idealo.

[.instruction]
Viewing the API log:

.  Go to **Data exchange » API log**.
.  Select the **IdealoDirektkauf** format from the **Process** drop-down menu.
.  Select a Date if you only want to see entries for a specific day.
.  Select the number of results per page.
.  Click on **Search** to display the results.
