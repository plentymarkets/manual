= Calendar
:lang: en
include::{includedir}/_header.adoc[]
:position: 100

The plentymarkets calendar is a fully integrated calendar module. It is located in the **Start » Calendar** menu of your plentymarkets system. The calendar allows you to make appointments and plan deadlines. This helps you organize your own and your employees' time and improve internal processes.

== Structure of the calendar

The calendar displays your appointments. In addition, it shows the appointments of any users whose calenders have been made visible to you and which you have activated in the **filter**.

You can choose to display the **Day**, **Week** or **Month**. In the top left corner of the calendar, the button **Today** is displayed. You can switch back to displaying the current day by clicking this button at any time.

The filter shows a list of all users for which you at least have read permissions. Place a check mark to display a user's appointments in your calendar.

== Setting a user as owner

Except for the <<basics/admin-tasks/manage-users#, user classes>> **API** and **Variable**, all classes are automatically owner of a calendar. For the **Variable** user class, rights need to be granted to use the calendar. No calendar is available for the user class **API**. The right assigned to a user can be deactivated in the **Owner** tab at any time.

[.instruction]
Activating a user as owner:

. Go to **System » Settings » User » Accounts**.
. Open the user.
. Click on the **Owner** tab.
. Place a check mark next to the option **Calendar**.
. *Save* the settings. +
→ The user can create and edit appointments.

Once a user is owner of a calendar, the new **Calendar** tab will be displayed and can be configured.

== Calendar settings

Proceed as described below to configure the calendar settings and rights.

[.instruction]
Configuring the calendar settings:

. Go to **System » Settings » User » Accounts**.
. Open the user.
. Open the **Calendar** tab.
. Pay attention to the explanations given in table 1 and carry out the settings as desired.
. *Save* the settings.

[cols="1,3"]
|====
|Setting |Explanation

|**Server address**; +
**Server path**; +
**Port**
|Automatically populated by the system. +
The server address and server path are required for configuring an external calendar tool such as Google Calendar.

|**New password**
|A password must be entered during setup of the calendar account. This password will be required for the external calendar tools. You can change the password at any time. +
**Note:** After saving, the password is no longer displayed and can not be retrieved. Make a note of the password and store it in a safe place. +
If you use an external calendar tool, you might need to reenter the password when the password is changed.

|**Repeat password**
|Reenter the new password for confirmation.

|**Description**
|The description is displayed within the information and settings for the calendar.

|**Right**
|Select which users can access this owner's calendar. Also select the type of right assigned to these users. Users must to be assigned at least read permission in order to see the calendar. +
**No access** = The user has neither read nor write permissions for this owner's calendar. +
**Read** = The user can view the calendar. +
**Write** = The user can edit appointments in the calendar.
|====

__Table 1: options in the **Calendar** tab__

== Creating and editing appointments

Proceed as described below to create, edit or delete appointments. Depending on your access rights, you might be able to create appointments for other users and edit their existing appointments.

=== Creating an appointment

You can only create appointments for users for which you have write permissions. You can not select users for which you do not have write permissions during creation of an appointment. Proceed as described below to create an appointment.

[.instruction]
Creating an appointment:

. Select a time or a time period. You can select a time by clicking on the field representing this particular point in time. Alternatively, you can select a time period by clicking and holding down the mouse button in the starting time field and then dragging the mouse to the ending time.
. Pay attention to the explanations given in table 2 and carry out the settings as desired.
. *Save* the settings.

[cols="1,3"]
|====
|Setting |Explanation

|**Title**
|Will be displayed in the calendar overview.

|**All day**
|Specifies that the appointment will last the entire day. This is especially recommended for appointments that are scheduled to last multiple days. In the daily and weekly view of the calendar, all appointments will be displayed in the top row. This helps to ensure that the calendar will not become cluttered.

|**From / To**
|The date and time are preselected but can be edited.

|**Type**
|Select the type of appointment from the drop-down list.

|**Calendar**
|Select the owner you want to create the appointment for.

|**Description**
|Enter a description, e.g. details regarding the appointment. This description is displayed in the appointment details and visible to all users with access rights to the calendar.
|====

__Table 2: appointment details__

=== Displaying and editing appointment details

Click on an appointment in order to see details about it. Click on the **X** in the upper right-hand corner of the **Appointment details** window to close the detailed information. You can edit all of the details for an existing appointment. To do so, proceed as follows.

[.instruction]
Editing an appointment:

. Click on the appointment that you would like to edit. +
→ The appointment details will be displayed.
. Make the desired changes.
. Click on the **save icon** to save the changes.

=== Deleting an appointment

There is also a delete function available in the appointment details. Proceed as described below to delete an appointment.

[.instruction]
Deleting an appointment:

. Click on the appointment that you would like to delete. +
→ The appointment details will be displayed.
. Click on the **delete icon**. +
→ The appointment will be deleted.

== Synchronizing the calendar with external calendar tools

You can use a link:https://de.wikipedia.org/wiki/CalDAV[CalDAV^] interface to synchronize your calendar with external calendar tools. For example, you can synchronize your calendar with tools such as Mac Calendar or iPad Calendar. The **Calendar** function needs to be activated in the user's profile as described above. This is done in the **Owner** tab. You need to know the settings for the user in order to carry out the configuration. The settings are saved in the user's **Calendar** tab. Refer to the external tool's documentation to find out how to synchronize your calendar with it.
