== Number formats

In this menu, you can save various settings related to **number formats**. For example, you can specify how many **decimal places** should be used for dimensions and weights.

== Configuration

Typical values are already saved by default.

image::basics/working-with-plentymarkets/basic-settings/assets/EN-Settings-Basic-Settings-Number-Formats01-SI.png[]

__Image 1: basic settings for **number formats**__

The following table includes detailed information and important notes:

[cols="a,a"]
|====
|Setting |Explanation

|Number format: decimal separator
|Select whether you would like __**prices in the online store**__ to be displayed with a __**comma**__ or __**period**__ as the decimal separator. +
**###,##** = Comma +
**###.##** = Period

|Number format: decimal places
|Select the number of possible decimal places for __**quantity entries**__. The quantity will be displayed on documents (such as invoices and delivery notes) the way that you selected. +
**No decimal place** = __**rounded quantities**__ will be displayed. +
**1** or **2 decimal places** = **__no more than the selected number of decimal places__** is displayed **__after the comma__**.

|Number format: Weights
|Select how many __**decimal places**__ should be displayed when the weight is given __**in kg**__. Alternatively, you can choose to display the __**weight in grams**__. +
**Important:****__If no decimal place__** is set, then items that **__weigh less than 1000 grams might be displayed with a weight of 0__**.

|**Number format: Content**
|Select the number of __**decimal places**__ for content. You can choose between 1 and 3 decimal places or you can select **no decimal place**. +
**Default setting**: 2

|Rounding: Distribution
|Specify how the __**percentages**__ should be rounded during the distribution. You can choose the options __**Half round up**__, __**round down**__ or __**round up**__. +
**Note**: If you select the setting __**Half round up**__, then the last number will automatically be **__rounded up__** (final digits 9 to 5) or __**rounded down**__ (final digit 4 to 1).
|====
