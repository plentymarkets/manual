Auf dieser Handbuchseite wird beschrieben, wie Sie Dokumentenvorlagen einrichten. Dokumentenvorlagen dienen zum Erzeugen von Dokumenten. Ohne Vorlagen können keine Dokumente erzeugt werden. Das Erzeugen der Dokumente erfolgt meist nach bestimmten Ereignissen. So wird ein Reparaturschein nur benötigt und erzeugt, wenn ein Artikel zur Reparatur eingegangen ist. In plentymarkets gibt es zwei Arten von Dokumenten, standortbezogene Dokumente und standortunabhängige Dokumente. Standortbezogen bedeutet, dass Sie für jeden Standort eine eigene Vorlage für das Dokument einrichten können. Standortunabhängig heißt, dass eine Vorlage für alle Standorte gilt. Alle Dokumentvorlagen können mehrsprachig angelegt werden. Außerdem erfahren Sie, wie Sie Ihr Geschäftspapier als PDF-Vorlage speichern und mit einer Dokumentenvorlage verknüpfen. Auch das Anlegen von Nummernkreisen wird erläutert.

[#10]

== PDF-Vorlage verknüpfen

Mit PDF-Vorlagen können Sie einen statischen Hintergund für Ihre Vorlagen speichern, sozusagen das Briefpapier Ihres Webshops. So kann eine PDF-Vorlage bereits Sie Ihre Firmendaten, Layout-Elemente gemäß dem Corporate Design Ihres Unternehmens sowie Ihr Logo enthalten. Einige dieser Angaben können Sie auch dynamisch auf einer Dokumentvorlage platzieren. Angaben, die Ihre PDF-Vorlage enthält, sollten Sie nicht noch einmal in der Vorlage platzieren.

[cols="a,a"]
|====
|Eigenschaft |Voraussetzung

|**PDF-Version**
|1.4 oder niedriger

|**Maximale Größe**
|160 kB
|====

Nutzen Sie z.B. ein Textverarbeitungsprogramm, um eine Geschäftsbriefvorlage inkl. Logo und allen Firmendaten zu erstllen. Erzeugen Sie aus diesem Dokument eine PDF-Datei.

image::_entwurfsseiten/entwurf-sina/assets/DE-Einstellungen-Standort-Dokumente-11.png[ ]

__Bild 11: PDF-Erzeugung in OpenOffice__

[IMPORTANT]
.PDF-Format bestimmt Format der Dokumentvorlage
====
Das Format des PDFs wird für die Dokumentvorlage übernommen. Wenn das PDF-Format DIN A4 ist, ist auch die Dokumentvorlage DIN A4. Wenn Sie DIN A5 als PDF-Format nutzen, ist auch die Dokumentvorlage DIN A5. Beachten Sie das Format für die Einstellungen der Dokumentvorlage, z.B. Spaltenbreiten.
====

[IMPORTANT]
.PDF-Version prüfen
====
Wenn die PDF-Vorlage nicht kompatibel ist, kann dies zu Fehlern bei der Erzeugung der Dokumente führen. Aktuell unterstützt plentymarkets die **PDF-Version 1.4** oder niedriger. Die PDF-Version können Sie prüfen, indem Sie eine PDF-Datei erzeugen und in den Dateieigenschaften nachsehen, in welcher PDF-Version die PDF-Datei erzeugt wurde. Sollte eine Version höher als 1.4 eingestellt sein, korrigieren Sie die Einstellung auf mindestens 1.4. Die maximale Dateigröße für PDF-Vorlagen ist 160 kB.
====

Gehen Sie folgendermaßen vor, um die erstellte PDF-Datei mit einer Dokumentenvorlage, z.B. der Vorlage **Rechnung**, zu verknüpfen:

image::_entwurfsseiten/entwurf-sina/assets/DE-Einstellungen-Standort-Dokumente-12.png[ ]

__Bild 12: PDF-Vorlagen verschiedener Sprachen und Zahlungsarten__

[.instruction]
PDF-Vorlage mit einer Dokumentenvorlage verknüpfen:

. Öffnen Sie das Menü **System » Aufträge » Dokumente**.
. Wählen Sie die Dokumentenvorlage, mit der Sie die PDF-Vorlage verknüpfen möchten.
. Wählen Sie eine **Sprache** aus dem Dropdown-Menü, z.B. **de** für Deutsch.
. Klicken Sie auf das Tab **PDF-Vorlage**.
. Klicken Sie auf das Tab **Neue Vorlage**.
. Wählen Sie ggf. eine **Zahlungsart**.
. Klicken Sie in der Zeile **Datei** auf **Durchsuchen**.
. Wählen Sie die PDF-Dokumentenvorlage.
. **Speichern** Sie die Einstellungen. +
→ Im Tab **Vorlagen** wird die Vorlage mit einer Angabe zur Dateigröße angezeigt.
. Klicken Sie im Tab **Vorlagen** auf **Vorschau**, um eine Vorschau der reinen PDF-Vorlage anzuzeigen.
. Wechseln Sie in das Tab **Vorlage**, um eine Vorschau der Dokumentvorlage mit dem PDF als Hintergrund anzuzeigen.
. Korrigieren Sie die PDF-Vorlage oder die Koordinaten der Dokument-Vorlage, falls sich Inhalte überschneiden. +
→ Wenn Sie Änderungen an der PDF-Vorlage vorgenommen haben, laden Sie das PDF erneut hoch und speichern es. Die vorige Vorlage wird dabei überschrieben.

Für jede Sprache kann je eine Vorlage hinterlegt werden.

Eine PDF-Vorlage für eine Zahlungsart benötigen Sie z.B., wenn für eine Zahlungsart andere Kontodaten gelten oder wenn Sie für ein bestimmtes Lieferland eine sprachlich angepasste Version verwenden möchten.

[IMPORTANT]
.Prinzip für die Auswahl einer PDF-Vorlage
====
Das System wählt die PDF-Vorlage anhand der Auftragssprache. Die Auftragssprache ergibt sich je nach Herkunft. Bei Aufträgen aus Ihrem Webshop, wird die Sprache in der der Kunde gekauft hat als Auftragssprache gespeichert. Wenn eine Auftrag von einem Marktplatz stammt, wird die Sprache des Lieferlandes als Auftragssprache gespeichert. Sobald die Sprache einmal für den Auftrag gespeichert wurde, basiert die Auswahl der PDF-Vorlage darauf und ändert sich auch nicht, wenn z.B. für den Kunden eine andere Sprache eingestellt wird. Wenn für einen Standort für eine Sprache keine PDF-Vorlage gefunden wird, sucht das System automatisch in folgender Reihenfolge nach übereinstimmenden PDF-Vorlagen:

* Standardsprache des Mandanten
* Systemsprache
* Standardstandort des Mandanten
* Sprache des Standorts +
Bei Rechnungen wird hier die Zahlungsart nicht mehr berücksichtigt.

Wenn gar keine PDF-Vorlage gefunden wird, erhält das Dokument einen weißen Hintergrund.
====

== Dokumentenvorlage einrichten

Die Einrichtung einer Vorlage für alle oben aufgeführten Dokumente erfolgt nach dem gleichen Prinzip. Für welche Dokumente Vorlagen eingerichtet werden können, sehen Sie in der Tabelle ###. Je nachdem ob es sich um eine standortübergreifende oder um eine standortbezogene Vorlage handelt, finden Sie die Vorlage in unterschiedlichen Menüs.

Beim ersten Öffnen einer Vorlage sind die meisten Vorlagen leer. In einigen Vorlagen, wie der Rechnungsvorlage, sind bereits einige Einstellungen voreingetragen. Diese Einstellungen können Sie bearbeiten und weitere Einstellungen vornehmen. Wenn keine Einstellungen voreingetragen sind, können Sie Einstellungen aus einer **<<_entwurfsseiten/entwurf-sina/entwurf-sina1#150,  Standard-Dokumentenvorlage übernehmen>>**. Bei welchen Vorlagen das Übernehmen von Standard-Einstellungen möglich ist, sehen Sie in Tabelle ###. Ergänzen Sie außerdem Hinweistexte, die durch Bedingungen gesteuert automatisch eingesetzt werden. Legen Sie für jede Sprache eigene Vorlagen an.

[IMPORTANT]
.Vorschau anzeigen ist ohne Dokument nicht möglich
====
Die Vorschau für ein Dokument wird erst angezeigt, wenn schon ein solches Dokument in plentymarkets erzeugt wurde, da sonst keine Daten für die Vorschau vorhanden sind. Nutzen Sie z.B. einen Testauftrag, um das Dokument Rechnung zu erzeugen und so die Vorschau anzuzeigen. Es werden immer die Daten des neusten Dokuments für die Vorschau genutzt.
====

[.instruction]
Vorlage einrichten:

. Öffnen Sie das Menü **System » Aufträge » Dokumente**. +
oder +
Öffnen Sie das Menü **System » Mandant » Mandant wählen » Standorte » Standort wählen » Dokumente**.
. Wählen Sie die Vorlage, die Sie bearbeiten möchten.
. Wählen Sie eine **Sprache** aus dem Dropdown-Menü.
. Nehmen Sie die Einstellungen gemäß den Erläuterungen auf dieser Handbuchseite vor.
. **Speichern** Sie die Einstellungen.

Bei Aufträgen aus Ihrem Webshop, wird die Sprache in der der Kunde gekauft hat als Auftragssprache gespeichert. Wenn eine Auftrag von einem Marktplatz stammt, wird die Sprache des Lieferlandes als Auftragssprache gespeichert. Das System prüft zunächst, ob in der Auftragssprache eine Vorlage vorhanden ist. Wenn eine Vorlage in der Sprache vorhanden ist, wird diese genutzt. Wenn für einen Standort keine Dokumentenvorlage in der benötigten Sprache gefunden wird, sucht Ihr plentymarkets System automatisch nach einer Ausweichvorlage. [#30]
=== Grundeinstellungen

Im Bereich **Grundeinstellungen** legen Sie die Schriftart, den Zeichsatz, die Währung des Dokuments etc. fest.

Neben der Standardeinstellung **ISO-8859-1** können Sie auch den Zeichensatz **Unicode** wählen. Beachten Sie, dass die Erzeugung von PDF-Dokumenten mit der Einstellung **Unicode** länger dauern kann.

[#40]
=== Seitenränder

Die Seitenränder sind wichtig für den automatischen Seitenumbruch und die Positionierung von Fließtext. Wenn Sie in Ihrer PDF-Vorlage eine Fußzeile haben, dann nutzen Sie den unteren Seitenrand, damit die Fußzeile nicht durch Inhalte des Dokuments überlagert wird.

[#50]
=== Positionierung der Pflichtelemente

Pflichtelemente sind Inhalte, die angezeigt werden müssen. Die Positionierung auf dem Dokument erfolgt über die Eingabe von Koordinaten. Die X-Achse verläuft von links nach rechts und die Y-Achse von oben nach unten. Die Nullpunkte liegen also links oben. In den Standardeinstellungen sind bereits Werte für die wichtigsten Parameter eingetragen. Diese Einstellungen können geändert werden. Ergänzen Sie die noch fehlenden Einträge.

[TIP]
.Beispiele für die Positionierung
====
Orientieren Sie sich für das Positionieren aller Felder an der linken oberen Ecke und beachten Sie die Seitenränder. Ein Feld, das üblicherweise links oben positioniert wird, ist das Adressfeld. In der Rechnung sind die Standardkoordinaten für das Adressfeld X = 21 (mm), Y = 54,5 (mm). +
Felder, die üblicherweise rechts positioniert werden, sind das Datum, die Auftrags-ID und die Kunden-ID. Diese IDs haben auf der X-Achse die Standardkoordinaten X = 150 (mm) für den Feldnamen und X = 173 (mm) für den Wert. Die Y-Koordinaten der IDs unterscheiden sich, damit die Felder und Werte untereinander liegen und sich nicht überlagern. +
Ein Feld, das häufig unten positioniert wird, ist die Seitenzahl. Die Standardkoordinaten für die Seitenzahl sind X = 21 (mm), Y = 286 (mm). Die Seitenzahl ist auf der Rechnung somit standardmäßig links unten.
====

[TIP]

== Nummernkreise

Für zahlreiche Dokumente stehen Nummernkreise zur Verfügung. Diese Nummernkreise verwalten Sie im Tab **Nummernkreis** eines Dokuments. Der Nummernkreis kann bis zu 5 Stellen haben und Zahlen, Buchstaben sowie Zeichen enthalten. Er wird automatisch vor der eigentlichen Nummer des Dokuments eingefügt.

=== Nummernkreise vergeben

Je nachdem für welche Standorte und Dokumente ein Nummernkreis verwendet werden soll, speichern Sie den gleichen Nummernkreis. Die Nummern werden dann übergreifend hochgezählt und jede Nummer pro Nummernkreis nur einmal vergeben. Wenn für jeden Standort und jedes Dokument ein eigener Nummernkreis verwendet werden soll, dann tragen Sie je einen anderen Nummernkreis ein. Sie haben außerdem die Möglichkeit auch pro Lieferland einen eigenen Nummernkreis zu vergeben. Als Nummernkreis können Sie z.b. ein Kürzel verwenden, das den Standort bezeichnet. +
Einige Möglichkeiten zur Verteilung von Nummernkreisen sind die folgenden:

* Ein Nummernkreis für mehrere Standorte und mehrere Dokumente
* Ein Nummernkreis für mehrere Standorte pro Dokument
* Ein Nummernkreis pro Standort
* Ein Nummernkreis pro Standort und Dokument
* Ein Nummernkreis pro Lieferland für alle Dokumente
* Ein Nummernkreis pro Lieferland und Standort
* Ein Nummernkreis pro Lieferland und Dokument
* usw.

[IMPORTANT]
.Nummernkreis pro Mandant
====
Nummernkreise müssen pro Mandant gepflegt werden und können nicht mandantenübergreifend verwaltet werden.
====

[TIP]
.Beliebige Zahl als Startwert
====
Der Startwert muss nicht zwingend die 1 sein. Wenn Sie z.B. bereits nummerierte Rechnungen im System abgelegt haben, können Sie eine Startnummer eintragen, die an die bestehenden Nummern anschließt. Wenn die Nummer über eine bestimmte Anzahl von Stellen verfügen soll, geben Sie den Startwert mit führenden Nullen an (z.B. 00001). Sind bereits Dokumente mit einer höheren Nummer im System, werden diese verwendet.
====

[IMPORTANT]
.Nummernkreis entfernen und ändern
====
Entfernen Sie einen Nummernkreis nicht mehr, sobald einmal PDF-Dokumente mit diesem Nummernkreis generiert wurden. Sie können Nummernkreise allerdings nachträglich ändern. Der geänderte Nummernkreis muss sich von allen bereits verwendeten Nummernkreisen unterscheiden. Das heißt, Sie müssen immer mindestens eine Stelle ändern oder den Nummernkreis um eine Stelle erweitern. Eine Stelle darf jedoch nicht entfernt werden. War ihr alter Nummernkreis z.B. 13IN, ist als neuer Nummernkreis 13I nicht zulässig, 13IN0 hingegen schon.
====

image::_entwurfsseiten/entwurf-sina/assets/DE-Einstellungen-Standort-Dokumente-13.png[ ]

=== EU-Lieferung

Bei Lieferungen in das europäische Ausland und vorliegender USt.-IdNr. wird die Rechnung automatisch netto ausgestellt. Aus steuerrechtlichen Gründen muss dabei die eigene USt.-IdNr. und die USt.-IdNr. des Kunden auf der Rechnung erscheinen. Außerdem muss ein Hinweistext abgedruckt werden.

Die USt.-IdNr. des Kunden ist als Datenfeld im Bereich **Positionierung der Pflichtelemente** verfügbar. Sie sollte im oberen Bereich der Rechnung mittels Koordinateneingabe positioniert werden.

Im Bereich **Optionale Elemente unter Artikelpositionstabelle** befindet sich das Datenfeld **Steuerrechtlicher Hinweis 1**. Dieses Datenfeld wird bei einer Lieferung in die EU bei vorliegender USt.-IdNr. ausgegeben. Ein Beispiel für diesen Hinweistext ist: "Ausfuhrlieferung gemäß §4 Nr. 1a in Verbindung mit §6 UStG."

[#240]
=== Ausfuhrlieferung

Ausfuhrlieferungen, d.h. Lieferungen in ein Land außerhalb der EU, werden ebenfalls netto erstellt. Für diesen Vorgang muss ebenfalls ein steuerrechtlicher Hinweistext ausgegeben werden. Für diesen Hinweistext steht das Datenfeld **Steuerrechtlicher Hinweis 2** zur Verfügung.

Die erforderlichen steuerrechtlichen Hinweise können individuell abweichen. Klären Sie den Wortlaut mit einem Steuerberater ab.

=== Differenzbesteuerte Artikelpositionen in Dokumenten

Den Steuersatz für die Differenzbesteuerung legen Sie im Menü **<<auftragsabwicklung/auftraege/buchhaltung#, System » Mandant (Shop) » Standard » Buchhaltung » Tab: Umsatzsteuersätze>>** fest. Für differenzbesteuerte Artikelpositionen und Summen gibt es in Dokumenten, wie der Rechnung, Mahnung oder Gutschrift, zwei Einstellungen:

* Zwischensumme für differenzbesteuerte Positionen (Bild 14)
* Hinweis zu den differenzbesteuerten Positionen (Bild 15)

== Nachbestellungen generieren

Um das erstellte Dokument für Nachbestellungen zu prüfen, legen Sie eine Nachbestellung an oder verwenden eine bestehende Nachbestellung. Dies ist wichtig, da bestimmte Elemente, z.B. Auftragsnotizen und die Nummer des Dokuments aus dem Tab **Nummernkreis**, nur in Verbindung mit einer Nachbestellung im Dokument ausgegeben werden.

Sie haben verschiedene Möglichkeiten, eine Nachbestellung zu erzeugen. Legen Sie z.B. Nachbestellungen im Menü link::/[**Waren » Bestellwesen » Nachbestellungen**] an.

image::_entwurfsseiten/entwurf-sina/assets/DE-Einstellungen-Auftraege-Dokumente-Nachbestellung-01.png[]

__Bild 1: Nachbestellung anlegen__

[.instruction]
Nachbestellung manuell erstellen:

. Öffnen Sie das Menü **Waren » Bestellwesen » Nachbestellungen**.
. Klicken Sie auf **Neue Nachbestellung**. +
→ Das Tab **Neue Nachbestellung** wird geöffnet.
. Wählen Sie die Parameter aus den Dropdown-Menüs.
. **Speichern** Sie die Einstellungen. +
→ Die neue Nachbestellung wird geöffnet und das Tab **Base** der Nachbestellung wird angezeigt.

[NOTE]
.Konfiguration und Anzeige
====
Im Tab **Base** der Nachbestellung konfigurieren Sie die Nachbestellung wie auf der Handbuchseite link::/[**Nachbestellungen**] beschrieben. Im Bereich **Grunddaten** klicken Sie bei der Option **Bestellen** auf das **Auge**, um das PDF-Dokument der Nachbestellung anzuzeigen (Bild 2, Pfeil). Nach einem Klick auf **Neu laden** wird nach Änderungen der Vorlage oder der Bestelldaten ein aktualisiertes Dokument angezeigt.

. Öffnen Sie das Menü **System » Aufträge » Dokumente » Pickliste**.
. Entscheiden Sie, welche Felder auf der Pickliste angezeigt werden sollen. Bestimmen Sie die Reihenfolge dieser Felder, indem Sie für jedes zu verwendende Feld eine **Positionsnummer** wählen.
. Tragen Sie unter **Name** ggf. eine eigene Bezeichnung für die verwendeten Felder ein.
. Bestimmen Sie die **Breite** für die Felder sowie die **Schriftgröße** und den **Schriftschnitt** für die jeweiligen Texte.

== Dokumentenarchiv

Im Dokumentenarchiv werden alle PDF-Dokumente abgelegt, die in Ihrem plentymarkets System erzeugt werden. Sie können sich die Dokumente für einen gewählten Zeitraum anzeigen lassen. Die Dokumente werden chronologisch gelistet. Außerdem können Sie in diesem Menü ein Sammel-PDF aus bis zu 200 Dokumenten erstellen.

Folgende Dokumente können im Archiv angezeigt werden:

* Rechnungen
* Externe Rechnungen
* Sammelrechnungen
* Lieferscheine
* Gutschriften
* Korrekturbelege
* Sammelgutschriften
* Auftragsbestätigungen
* Angebote
* Nachbestellungen
* Mahnungen
* Reparaturscheine
* Abhollieferungen
* Rücksendescheine
* Gelangensbestätigungen
* SEPA-Lastschriftmandate
* EBICS INI-Brief
* Wareneingangsbeleg (Lagerschein)
* Abrechnungsberichte

image::_entwurfsseiten/entwurf-sina/assets/DE-Auftraege-Dokumentenarchiv-01-SI.png[]

__Bild 1: Das **Dokumentenarchiv**__

[.instruction]
Dokumente listen und anzeigen:

. Öffnen Sie das Menü **Aufträge » Dokumentenarchiv**.
. Wählen Sie aus dem Dropdown-Menü **Dokument** den Dokumententyp, den Sie suchen möchten.
. Wenn Sie ein bestimmtes Dokument suchen, tragen Sie unter **Nummer** die Dokumentennummer ein.
. Wählen Sie unter **Datum** den Zeitraum für die Dokumente.
. Klicken Sie auf **Suche**. +
→ Die gefundenen Dokumente werden chronologisch aufgelistet.
. Klicken Sie auf die Auftrags-ID, um den Auftrag des Dokuments zu öffnen.
. Klicken Sie auf den Link unter **Datei**, um das PDF-Dokument zu öffnen.

[NOTE]
.Zugehörigen Auftrag öffnen:
====
Klicken Sie auf eine **Auftrags-ID**, um den Auftrag der jeweiligen Rechnung zu öffnen.
====

[NOTE]
.PDF-Dokument öffnen:
====
Klicken Sie auf den jeweiligen Link unter **Datei**, um das jeweilige PDF-Dokument zu öffnen.
====

[.subhead]
Sammel-PDF erzeugen
Im Bereich **Gruppenfunktionen** haben Sie die Möglichkeit, PDF-Dokumente für die Funktion **Sammel-PDF** auszuwählen. Mit dieser Funktion werden ausgewählte Dokumente in einem PDF-Dokument zusammengefasst.

[IMPORTANT]
.Sammel-PDF beinhaltet maximal 200 Dokumente
====
Sie können maximal 200 Dokumente in einem PDF-Dokument zusammenfassen.
====

[.instruction]
Sammel-PDF erzeugen:

. Öffnen Sie das Menü **Aufträge » Dokumentenarchiv**.
. Wählen Sie die Parameter, anhand derer Sie Dokumente suchen möchten, wie oben beschrieben.
. Klappen Sie den Bereich **Gruppenfunktionen** auf.
. Aktivieren Sie die Dokumente, die Sie zu einem PDF-Dokument zusammenfassen möchten.
. Klicken Sie auf die Funktion **Sammel-PDF**.
. Klicken Sie auf **Ausführen**, um die PDF-Datei zu erzeugen. +
→ Die PDF-Datei wird in einem separaten Fenster geöffnet.

[WARNING]
.
====
Sie können **maximal 200 Dokumente** in einem PDF-Dokument zusammenfassen.
====
