Fulfillment beschreibt die einzelnen Schritte der Auftragsabwicklung, wie die Erzeugung von Rechnungen oder den Druck von Etiketten. Es wird auch beschrieben, wie Sie automatische Statuswechsel einstellen, die dann beim Durchlauf des Fulfillments automatisch ausgeführt werden. Diese Seite beschreibt die Funktionen der verschiedenen Bereiche des Fulfillments, die in den Untermenüs einstellbar sind. Das Ziel ist, Ihnen zu zeigen, wie Sie Ihren link::/[**Fulfillmentprozess möglichst effektiv gestalten**]. Dadurch gewährleisten Sie einen schnellen und somit optimalen Service, was im Onlinehandel enorm wichtig ist und u.a. dazu führt, dass aus Gelegenheitskäufern in Ihrem Webshop Stammkunden werden.

Diese Handbuchseite beschreibt die Konfiguration der wichtigsten Untermenüs im Menü **Aufträge » Fulfillment** und umfasst die Schritte, die ab der **Freigabe zum Warenversand** bis zum tatsächlichen **Warenausgang** durchlaufen werden. Die folgende Beschreibung ist als Beispiel zu verstehen, wie Sie die Vorgänge sinnvoll konfigurieren und einfach und schnell abarbeiten können.

Erfasst werden **Aufträge ab dem Status 5**. Bei diesen Aufträgen wurde bereits ein Zahlungseingang gebucht (Wechsel von Statusbereich 3 in 4) und außerdem ist die Ware verfügbar (Wechsel von Statusbereich 4 in 5).

Sie haben alternativ die Möglichkeit, diese Schritte weiter zu automatisieren indem Sie für bestimmte Schritte **<<auftragsabwicklung/auftraege/einstellungen/ereignis-aktionen#, Ereignisaktionen  >>** einrichten. Diese Seite enthält Beispiele dazu. Weiterhin können Sie Bereiche des Fulfillments über einen **link::/[Versanddienstleister]**, z.B. DHL Intraship, oder z.B. mit dem **<<basics/automatisierung/plentymarkets-client#, plentymarkets Client  >>** automatisieren.

== Vorbereitung

Damit Ihre Fulfillmentprozesse reibungslos ablaufen, müssen Sie die nachfolgend beschriebenen Bereiche prüfen und bei Bedarf bearbeiten.

=== Auftragsstatus anlegen

Legen Sie zunächst im Menü <<auftragsabwicklung/auftraege/einstellungen/auftragsstatus#, **Einstellungen » Aufträge » Auftragsstatus**  >> für jeden Schritt einen **Unterstatus** an, in den Aufträge dann jeweils geändert werden (Tabelle 1). Wir verwenden im Beispiel in Bild 1 die Standardbezeichnungen des Systems. Sie können diese Bezeichnungen auch ändern.

|Wählen, wie viele **Aufträge pro Durchgang** bearbeitet werden sollen. +
Es kann pro Durchgang höchstens **das maximal einstellbare Limit** an Aufträgen bearbeitet werden. Wenn die Anzahl der Aufträge, die bearbeitet werden sollen, größer ist, als das höchste einstellbare Limit, muss der Prozess entsprechend oft wiederholt werden. |**Ändere Auftragsstatus**
|Jeweils den Status wählen, in den die Aufträge nach dem jeweiligen Schritt wechseln sollen, siehe auch Tabelle 1. +
Der Statuswechsel ist wichtig, damit für bereits bearbeitete Aufträge ein Prozess nicht noch einmal durchgeführt wird. +
Wird keine Änderung gewünscht, z.B. wenn kein Prozess mehr folgt und der Auftrag im aktuellen Status bleiben kann, die Option **Keine Änderung** wählen.

== Lieferscheine erzeugen

Für die Erstellung der Lieferscheine konfigurieren Sie das Menü so, dass alle Aufträge erfasst werden, für die bereits Rechnungen erzeugt wurden und die sich im Status **Rechnungen erzeugt** befinden.

[TIP]
.Tipp: Keine Lieferscheine?
====
Wenn Sie keine Lieferscheine benötigen oder z.B. aus Kostengründen darauf verzichten möchten, lassen Sie diesen Schritt aus. Holen Sie diesbezüglich ggf. rechtlichen Rat ein.

== Packliste erzeugen

Die Packliste enthält u.a. Informationen zur Versandart, damit Sie die Aufträge entsprechend verpacken können. Wenn Sie z.B. == Lieferanschriftenliste

Wenn Sie Pakete über einen Versanddienstleister versenden, benötigen Sie eine **Lieferanschriftenliste**. Es handelt sich dabei um einen **Export mit Sendungsinformationen**, den Sie dem jeweiligen Versanddienstleister anschließend bereitstellen, so dass dieser Ihnen wiederum gültige **Versandetiketten** für Ihre Pakete zur Verfügung stellen kann.

In diesem Menü exportieren Sie die **Lieferanschriften Ihrer Aufträge** in eine Datei, die Sie für die Software Ihres Versanddienstleisters benötigen.

== Adressetiketten

Im Gegensatz zu Versandetiketten können Sie für **link::/[Adressetiketten]** in plentymarkets genau festlegen, welche Informationen an welcher Stelle enthalten sein sollen. Da Sie Adressetiketten nicht vorgefertigt von einem Versanddienstleister erhalten, ist hier weder das Porto bereits abgerechnet, noch eine Sendungsnummer enthalten. Die Verwendung von Adressetiketten eignet sich besonders für **Warensendungen**, z.B. den Versand von Büchern, die Sie durch den individuellen Aufdruck **Büchersendung** u.U. günstiger versenden können als über das Gewicht des Pakets. Sie können in plentymarkets die benötigten Adressetiketten konfigurieren, eine PDF-Datei erzeugen und mit dieser dann Etiketten ausdrucken.

[NOTE]
