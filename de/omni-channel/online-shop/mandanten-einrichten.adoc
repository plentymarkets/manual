= Mandanten
:lang: de
include::{includedir}/_header.adoc[]
:keywords: Webshop, Mandant, Standard
:position: 20
:nav-alias: Mandanten einrichten

Im Menü *Einstellungen » Mandant (Shop)* verwalten Sie Ihren <<10, *Standard-Mandanten* >> sowie zusätzliche Mandanten (Shops). Hier verknüpfen Sie zudem mehrere <<omni-channel/online-shop/mandanten-einrichten/webshop, *Webshops* >> mit einem Backoffice/Administrationsbereich. Mit dieser Funktionalität können Sie weitere Mandanten (Shops) parallel zu Ihrem Standard-Webshop einrichten. In einem plentymarkets System können bis zu 30 eigenständige Webshops (Mandant-Shops) eingerichtet und verwaltet werden. Diese Webshops sind datentechnisch geschlossene, eigenständige Systeme. Es ist somit möglich, über ein plentymarkets System mehrere unterschiedliche Geschäftsbereiche zu realisieren.

[#10]
== Standard-Mandanten einrichten

Im Menü *Einstellungen » Mandant (Shop) » Standard* finden Sie die verschiedenen Grundeinstellungen zur Konfiguration Ihres <<omni-channel/online-shop/mandanten-einrichten/webshop, *Webshops*  >>. Neben der Konfiguration des <<crm/e-mails-versenden#700, *E-Mail-Dienstes* >> können Sie hier verschiedene Einstellungen zu Ihrem Standard-Webshop hinterlegen. Von besonderer Bedeutung sind hier die Einstellungen zur <<omni-channel/online-shop/cms-syntax#fallbeispiele-seo-guide, *Suchmaschinenoptimierung*  >>, mit denen Sie Ihren Webshop im Internet optimal auffindbar machen. Darüber hinaus richten Sie die Anzeige von <<artikel/kategorien#, *Kategorien*  >>, den <<omni-channel/online-shop/mandanten-einrichten/mein-konto#, *Mein Konto*  >>-Bereich und den <<omni-channel/online-shop/mandanten-einrichten/bestellvorgang#, *Bestellvorgang*  >> Ihres Webshops ein. Auch Informationen zu den Grundeinstellungen Ihrer <<omni-channel/online-shop/mandanten-einrichten/artikelansichten#, *Artikelansichten*  >> finden Sie hier.

[TIP]
.Tipp
====
Jeder weitere Webshop bzw. Mandant (Shop) wird in einer eigenen Menüstruktur analog zu diesem Menü konfiguriert. Die Menübezeichnung wird entsprechend der Bezeichnung des Webshops automatisch vom System vorgenommen.
====

[#20]
=== Grundeinstellungen vornehmen

Im Menü *Einstellungen » Mandant (Shop) » Standard » Grundeinstellungen* werden die IDs Ihres Webshops angezeigt sowie weitere Grunddaten, wie der Webstore-Typ und die Domain. Der hinterlegte Name des Webshops ist hier änderbar.

[WARNING]
.Benutzer werden ausgeloggt
====
Wenn Sie in diesem Menü Änderungen vornehmen, werden andere Benutzer automatisch ausgeloggt und müssen sich neu einloggen, damit die Änderungen auch für diese Benutzerprofile greifen. Kündigen Sie Änderungen daher ggf. weiteren Benutzern an.
====

Nehmen Sie die Konfiguration gemäß der nachfolgenden Tabelle vor.

.Grundeinstellungen vornehmen
[cols="a,a"]
|====
|Einstellung |Erklärung

|*Webstore-ID*
|Die Webstore-ID ist die ID des Webshop bzw. Mandant (Shop). Die Webstore-ID eines Mandant (Shop) wird in den Grundeinstellungen des betreffenden Shops angezeigt. +
Standard-Webshop = *0* +
Erster Mandant (Shop) = *1* +
Zweiter Mandant (Shop) = *2* +
usw. +
Ein externer Shop erhält ebenfalls eine Webstore-ID.

|*plenty-ID*
|Hier wird die ID des plentymarkets-Systems angezeigt.

|*Webstore-Typ*
|Anzeige des Webstore-Typ, *plentymarkets* oder *Shopware*.

|*Domain*
|Hier wird die URL der Domain des Webshops angezeigt. Über das *Auge* wird der Webshop aufgerufen.

|*Name*
|Name des Webshops.

|*Standard-Sprache*
|Die Standard-Sprache des Webshops wählen.

|*Standard-Standort*
|Den Standort wählen, der als Standard gelten soll. Dieser wird dann z.B. auch im Bestellvorgang im Webshop verwendet, was u.a. die Anzeige der MwSt. beeinflusst. Ein Standort, der hier als Standard hinterlegt wird, ist nicht löschbar.

|*eBay-Konto*
|Das oder die eBay-Konten wählen, die mit diesem Shop verknüpft werden sollen. Die Aufträge dieses eBay-Kontos werden automatisch diesem Shop zugeordnet. +
*Wichtig*: Wenn dasselbe eBay-Konto bei zwei verschiedenen Shops ausgewählt werden soll, werden die Aufträge dem Shop mit der niedrigeren ID zugeordnet.
|====

== Weitere Mandanten einrichten

Die Anwendung zusätzlicher Mandanten (Shops) bietet sich an, wenn sehr verschiedenartige Produktbereiche über unterschiedliche Domains und Webshops angeboten werden sollen. Die Verwaltung der Aufträge und Artikel läuft dabei über ein zentrales System. Für die einzelnen Webshops kann bestimmt werden, welches Design angezeigt werden soll und welche Artikel angeboten werden sollen. Die Abwicklung der Aufträge inklusive Rechnungsdruck und E-Mail-Service läuft zentral über ein System. Es ist dabei eine Unterscheidung der <<auftragsabwicklung/auftraege/auftragsdokumente#200, *Nummernkreise* >> pro Mandant (Shop) und pro Lieferland möglich.

== Mandanten einrichten

Nachdem der Mandant (Shop) aktiviert wurde, nehmen Sie die Einstellungen unter *Einstellungen » Mandant (Shop)* im Menü des Mandanten vor. Die Einstellungsmöglichkeiten entsprechen weitgehend den Einstellungen für Standard-Webshops. Beachten Sie daher die Erläuterungen im Kapitel <<50, *Standard-Mandanten einrichten*  >>.

[IMPORTANT]
.Menübezeichnung
====
Das Menü eines Mandanten erhält den Namen des Shops. Sie finden das Hauptmenü eines Mandanten im Menü *Einstellungen » Mandant (Shop)* unterhalb des Menüs *Standard* (= Menü Ihres Standard-Webshops). Dort werden die Menüs weiterer Mandanten (Shops) angeordnet (Bild 1).
====

image::assets/DE-Einstellungen-Mandant-Shop-01.png[]

_Bild 1: Mandanten (Shops); Übersicht_

In der folgenden Tabelle finden Sie Informationen zu den Einstellungen, die sich von den Webshop-Standard-Einstellungen unterscheiden.

.Abweichende Einstellungen für Mandanten (Shops)
[cols="a,a"]
|====
|Eintrag/Einstellung |Erläuterung

2+|*Grundeinstellungen*

|*Webstore-ID*
|Webstore-ID des Mandanten. Die ID ist eindeutig und nicht änderbar. Mehrere Mandanten können anhand dieser Webstore-ID unterschieden werden. Der Standard-Webshop hat die ID *0*, Mandanten erhalten ab ID 1 fortlaufende IDs.

|*Domain*
|Die *<<basics/arbeiten-mit-plentymarkets/plentymarkets-konto#domain, Domain  >>* des Mandanten (Shops). Eine Domain bezeichnet in der Regel eine Internetadresse in Form eines Namens sowie einer Erweiterung, d.h. der Top Level Domain, beides getrennt durch einen Punkt. +
Wenn ein *<<basics/arbeiten-mit-plentymarkets/plentymarkets-konto#ssl-zertifikat, SSL-Zertifikat  >>* angelegt wurde, wird die URL des Mandanten mit *https* statt *http* angezeigt. +
*_Tipp:_* Über das *Vorschau-Symbol* eine Ansicht des Mandanten (Shops) aufrufen.

|*Standard-Sprache*
|Die Sprache für den Mandanten (Shop) wählen.

|*eBay-Konto*
|Zeigt die eBay-Konten an, die unter *<<omni-channel/multi-channel/ebay#65, Einstellungen » Märkte » eBay » Konten >>* angelegt wurden. Wählen, welches eBay-Konto für den Mandanten verwendet werden soll (Mehrfachauswahl mit *Strg + Mausklick*).

2+|*Menü <<omni-channel/online-shop/mandanten-einrichten/webshop#infoseiten, Webshop » Seiten  >>*

|*Seite Bankdaten*
|Wählen, welche Seite die *<<auftragsabwicklung/payment/bankdaten-verwalten#, Bankdaten  >>* enthält. Diese Seite wird als Teil der Auftragsstatus-Seite im Webshop angezeigt. +
*_Wichtig:_* Wird diese Einstellung verwendet, unter *<<auftragsabwicklung/payment/zahlungsarten-verwalten#, Einstellungen » Aufträge » Zahlung » Zahlungsarten  >>* bei den Zahlungsarten *Rechnung* und *vorab Überweisung/Vorkasse* die Option *Bankdaten anzeigen* wählen (Häkchen setzen).

2+|*Menü <<omni-channel/online-shop/mandanten-einrichten/bestellvorgang#, Bestellvorgang » Grundeinstellungen  >>*

|*Standard-Lieferland als Firmensitz verwenden*
|*Ja* wählen, wenn die steuerbezogenen Einstellungen, z.B. die MwSt., im System an das eingestellte Standard-Lieferland angepasst werden soll. +
_*Wichtig:*_Für das Standard-Lieferland müssen im Menü *<<auftragsabwicklung/auftraege/buchhaltung#, Einstellungen » Mandant (Shop) » Mandant öffnen » Buchhaltung  >>* Steuersätze hinterlegt sein. +
*_Beispiel_*: Ein Händler mit Sitz in Deutschland hat viele Kunden in Frankreich und richtet für diesen Markt einen Mandanten (Shop) ein. Er wählt *Frankreich* als *Standard-Lieferland*. Mit der Einstellung *Ja* bei *Standard-Lieferland als Firmensitz verwenden* wird nun Frankreich als *System-Land* für diesen Mandanten übernommen. Alle damit verbundenen Prozesse werden jetzt behandelt, als würde sich der Firmensitz in Frankreich befinden.

2+|*Menü <<crm/e-mails-versenden#, E-Mail  >>*

|*E-Mail-Einstellungen*
|Soll für einen Mandanten (Shop) eine gegenüber dem Webshop abweichende E-Mail-Konfiguration verwendet werden, diese im Untermenü <<crm/e-mails-versenden#, *E-Mail*  >> einrichten. Wenn das gleiche Konto verwendet werden soll, die Einstellungen aus der Konfiguration des Standard-Webshops übernehmen.
|====

== Artikel auf Mandanten verteilen

Welche Artikel in welchem Mandanten (Shop) angeboten werden, steuern Sie über die Verknüpfung mit Kategorien. In jedem Mandanten werden nur die Kategorien angezeigt, die im Menü *<<artikel/artikel-verwalten#230, Artikel » Kategorien >>* für diesen Mandanten aktiviert werden.

[.instruction]
Kategorien aktivieren:

. Öffnen Sie das Menü *Artikel » Kategorien*.
. Öffnen Sie die Kategorie, die Sie für einen oder mehrere Mandanten (Shops) aktivieren möchten.
. Markieren Sie die Mandanten (Shops), in denen Sie Artikel dieser Kategorie anbieten möchten.
. Wählen Sie ggf. die Option *Sichtbarkeit auf Unterkategorien übertragen*, um die Auswahl auf alle Unterkategorien der Kategorie zu übertragen.
. Klicken Sie auf *Speichern*, um die Einstellungen zu sichern. +
→ Die Kategorie ist in den gewählten Mandanten (Shops) sichtbar.

[NOTE]
.Sichtbarkeit von Artikeln festlegen
====
Die Verfügbarkeit und Sichtbarkeit von Artikeln stellen Sie im Tab *<<artikel/artikel-verwalten#230, Einstellungen >>* der Variante ein.
====

Erfahren Sie mehr zur Verknüpfung von Artikeln mit Kategorien im folgenden Videotutorial:

[.linkBox]
link:https://www.plentymarkets.eu/a-3147/[Artikel mit Kategorien verknüpfen, window="_blank"]

[#40]
== Mehrere Mandanten mit einem Webshop verknüpfen

Die Verteilung der Kategorien auf mehrere Mandanten kann auch angewendet werden, wenn nur ein Webshop eingerichtet ist. Dieses Verfahren ist empfehlenswert, wenn ein Artikelkategoriebaum mit der Zeit stark angewachsen ist und der Webshopkunde den Überblick verlieren könnte. Mithilfe der Mandanten kann schnell eine weitere Kategorieebene eingefügt werden, um so den Kategoriebaum übersichtlicher zu gestalten.

[NOTE]
.Mandanten mit Webshop verknüpfen
====
Legen Sie Mandanten nach Themen sortiert an. Aktivieren Sie dann die Kategorien je nach gewünschtem Sortiment des Shops. Pro Shop werden nur die Kategorien des Mandanten angezeigt, die im Menü *<<artikel/kategorien/kategorien-bearbeiten#2, Artikel » Kategorien >>* gewählt wurden. Zusätzlich zur Standard-Kategorienavigation (Bild 2, lila Pfeil) wird das *<<omni-channel/online-shop/cms#webdesign, Webshop-Design  >>* um eine weitere Navigation, beispielsweise am Seitenkopf, erweitert (Bild 2, grüner Pfeil). Dort werden die Mandanten verlinkt. Die Navigationsleiste zu den einzelnen Mandanten ist in diesem Fall über die Template-Funktion *Link_Store(X)* verfügbar. Anstelle des X geben Sie die ID des Shops an.
====

image::assets/DE-Einstellungen-Mandant-Shop-02.png[]

_Bild 2: Zusätzliche Navigationsleiste im Webshop (grüner Pfeil)_

[TIP]
.Beispiel
====
Bild 2 zeigt ein Beispiel für einen Webshop (schwitzen.com), mit dem ein Mandant (organicum.de) verknüpft wurde. In der Navigationsleiste oben ist der Mandant (Shop) organicum.de verlinkt. Angenommen, dieser Mandant (Shop) hat im plentymarkets-Backend die ID 2. Der Link zu organicum.de sieht im Design dann wie folgt aus: *&lt;a href="{% Link_Store(2) %}"&gt;organicum.de&lt;/a&gt;*.
====
