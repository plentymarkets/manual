== Allgemeines zur Lager-Pickliste

Mit plentymarkets können Sie Lager-Picklisten als PDF-Dateien generieren. Im Gegensatz zur Pickliste ist bei der Lager-Pickliste eine Zuordnung zum Auftrag bzw. zur Rechnung möglich. So können die Artikel bei der Kommissionierung direkt zugeordnet und anschließend zum Verpacken gegeben werden.

== Artikeldaten darstellen

Im Menü **System » Aufträge » Dokumente » Lager-Pickliste** bestimmen Sie, welche Artikeldaten Ihre Lager-Picklisten enthalten, z.B. **Artikel-ID**, **Menge**, **Lagerort** oder **Rechnungsnummer**. Außerdem können Sie festlegen, an welchen Positionen und in welcher Reihenfolge diese Daten auf der Lager-Pickliste erscheinen.

.Optionen im Menü **Lager-Pickliste**
image::auftragsabwicklung/auftraege/auftragsdokumente/assets/DE-Einstellungen-Auftraege-Dokumente-Lager-Pickliste-01.png[]

[IMPORTANT]
.Felder anzeigen bzw. verbergen
====
Damit ein Feld angezeigt wird, muss für das jeweilige Feld eine Positionsnummer ausgewählt sein. Wenn Sie für ein Feld keine Anzeige auf der Lager-Pickliste wünschen, wählen Sie für die Positionsnummer die Einstellung **Pos**.
====

[.instruction]
Artikeldaten darstellen:

. Öffnen Sie das Menü **System » Aufträge » Dokumente » Lager-Pickliste**.
. Entscheiden Sie, welche Felder auf der Lager-Pickliste angezeigt werden sollen. Bestimmen Sie die Reihenfolge dieser Felder, indem Sie für jedes zu verwendende Feld eine **Positionsnummer** wählen.
. Tragen Sie unter **Name** eine eigene Bezeichnung für die verwendeten Felder ein.
. Bestimmen Sie die Breite für die Felder sowie die Schriftgröße und den Schriftschnitt für die Texte.
. Nehmen Sie weitere Einstellungen gemäß <<tabelle-Auftraege-Optionen-Lager-Pickliste>> vor.
. *Speichern* Sie die Einstellungen.

.Pflichteinstellung

== PDF-Vorlage hochladen

Eine Vorlage wird für das "Briefpapier" Ihres Shops benötigt, also eine Seite, die Ihre Firmendaten, Layoutelemente gemäß dem Corporate Design Ihres Unternehmens sowie Ihr Logo enthält. Diese Vorlage wird für das System im PDF-Format benötigt. Mit plentymarkets können Sie eine Dokumenten-Vorlage im PDF-Format für die Lager-Pickliste verwenden. Gehen Sie wie im Folgenden beschrieben vor, um eine PDF-Vorlage hochzuladen.

[.instruction]
PDF-Vorlage hochladen:

. Öffnen Sie das Menü **System » Aufträge » Dokumente » Lager-Pickliste**.
. Klicken Sie auf das Tab **PDF-Vorlage**.
. Klicken Sie auf das Tab **Neue Vorlage**.
. Klicken Sie auf **Durchsuchen**.
. Wählen Sie eine PDF-Datei als Vorlage aus.
. *Speichern* Sie die Einstellungen. +
→ Die PDF-Vorlage erscheint nun im Tab **Vorlagen**. Sie können dort eine Vorschau sehen oder die PDF-Vorlage wieder löschen.

[TIP]
.Tipp
====
Wenn Sie z.B. eine PDF-Vorlage mit "Header" verwenden und Sie diesen Header oberhalb der Lager-Pickliste positionieren möchten, müssen Sie ggf. den hinterlegten Wert im Feld **Seitenrand oben** noch anpassen (siehe Tabelle 1).
====

Weitere Details zu Dokumenten-Vorlagen finden Sie auf der Handbuchseite **<<auftragsabwicklung/auftraege/auftragsdokumente#, Dokumente>>**.

== Allgemeines zur Lager-Pickliste

Im Menü **Aufträge » Fulfillment » Lager-Pickliste** haben Sie die Möglichkeit, eine Lager-Pickliste zu erzeugen sowie dabei eine Statusänderung einzustellen. Die Lager-Pickliste wird als PDF-Datei erzeugt. Im Gegensatz zur Pickliste ist bei der Lager-Pickliste eine Zuordnung zum Auftrag sowie zur Rechnung möglich. So können die Artikel bei der Kommissionierung direkt zugeordnet und anschließend zum Verpacken gegeben werden. Im Unterschied zur allgemeinen Pickliste ist hier die Wahl eines bestimmten Lagers bindend.

== Lager-Pickliste konfigurieren oder ändern

Prüfen Sie die Konfiguration der Dokumentenvorlage in den Einstellungen der Lager-Pickliste, bevor Sie eine Lager-Pickliste erzeugen.

Der Link **PDF-Pickliste ändern** im Menü Lager-Pickliste führt zu den Einstellungen im Menü **System » Aufträge » Dokumente » <<auftragsabwicklung/auftraege/auftragsdokumente/lager-pickliste#, Lager-Pickliste>>**, in dem die Vorlage für das Dokument und die PDF-Vorlage für die Lager-Pickliste konfiguriert werden.

== Lager-Pickliste erzeugen

Um eine Lager-Pickliste zu erzeugen, gehen Sie vor wie nachfolgend beschrieben. +
Nachdem Sie die Einstellungen vorgenommen haben, prüfen Sie zunächst das Ergebnis (PDF-Datei). Korrigieren Sie die Einstellungen bei Bedarf, bis Sie mit dem Ergebnis zufrieden sind.

[.instruction]
Lager-Pickliste erzeugen:

. Öffnen Sie das Menü **Aufträge » Fulfillment » Lager-Pickliste**.
. Wählen Sie die Einstellungen anhand der Erläuterungen in <<tabelle-Fulfillment-Optionen-Lager-Pickliste>>.
. Klicken Sie auf das **Zahnrad**, um die Pickliste zu erzeugen.


[[tabelle-Fulfillment-Optionen-Lager-Pickliste]]
.Optionen im Untermenü Lager-Pickliste
[cols="1,3"]
|====
|Einstellung |Erläuterung

|**Auftragsstatus**
|Status der Aufträge wählen, die für die Lager-Pickliste berücksichtigt werden sollen.

|**Eigner**
|Einen Eigner wählen, für dessen Aufträge die Dokumente erzeugt werden sollen oder die Option **ALLE**, wenn für alle Eigner die Dokumente erzeugt werden sollen.

|**Mandant (Shop)**
|**Standard** = nur Standard-Webshop +
**ALLE** = Webshop und Mandanten

|**Lager**
|Ein Lager oder die Option **ALLE** wählen.

|**Auftragslimit**
|Wählen, wie viele Aufträge pro Durchgang bearbeitet werden sollen. Einstellbar ist ein Wert zwischen 5 und 300. +
Es kann pro Durchgang höchstens das maximal einstellbare Limit an Aufträgen bearbeitet werden. Wenn die Anzahl der Aufträge, die bearbeitet werden sollen, größer ist, als das höchste einstellbare Limit, muss der Prozess entsprechend oft wiederholt werden. Je geringer das eingestellte Limit, desto optimaler ist die Systemleistung.

|**Rechnungen anhängen**
|Wählen Sie die Option **Ja**, um im Anschluss an die Lager-Pickliste die Rechnungen nach Auftrags-ID sortiert in die PDF-Datei einzufügen.

|**Rechnungen pro Gruppierung**
|Wählen, wie viele Rechnungen pro Gruppierung ausgegeben werden sollen.

|**Ändere Auftragsstatus**
|Status wählen, in den die Aufträge nach dem Erzeugen der Lager-Pickliste wechseln sollen. +
Der Statuswechsel ist wichtig, damit für bereits bearbeitete Aufträge ein Prozess nicht noch einmal durchgeführt wird.
|====
