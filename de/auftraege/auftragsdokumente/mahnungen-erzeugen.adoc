= Mahnungen
:lang: de
include::{includedir}/_header.adoc[]
:keywords: Mahnung, Mahnungen erzeugen, Auftragsabwicklung, Auftragsdokumente
:position: 30

In plentymarkets können Sie Mahnungen als PDF-Dateien generieren, um diese z.B. zu drucken oder Ihren Kunden als E-Mail Anhang zukommen zu lassen. +
Eine Mahnung können Sie nach Ablauf der Zahlungsfrist verwenden, um Ihre Kunden an die ausstehende Zahlung zu erinnern.

[#100]
== Dokumentenvorlage für Mahnungen einrichten

Bevor Sie Mahnungen generieren können, muss eine Dokumentenvorlage eingerichtet werden. Dies geschieht im Menü **System » Mandant » Mandant wählen » Standorte » Standort wählen » Dokumente**. +
Weitere Informationen und Anleitungen zur Einrichtung der Dokumentenvorlage finden Sie auf der Handbuchseite <<auftraege/auftragsdokumente#, Auftragsdokumente>>.

[#200]
== Einzelne Mahnung erzeugen

Eine einzelne Mahnung können Sie in plentymarkets bei Bedarf manuell direkt am Auftrag erzeugen.

[.instruction]
Einzelne Mahnung erzeugen:

. Öffnen Sie das Menü **Aufträge » Aufträge bearbeiten**. +
→ Die Auftragsübersicht öffnet sich.
. Wählen Sie durch Klick auf die **Auftrag-ID** den Auftrag aus, für den eine Mahnung erzeugt werden soll. +
. Wechseln Sie in das Tab **Belege**. +
. Wählen Sie im Drop-Down-Menü **Beleg erstellen** das Dokument **Mahnung** aus. +
. Nehmen Sie die Einstellungen anhand <<tabelle-auftraege-belege-mahnung>> vor. +
. **Speichern** Sie die Einstellungen, um die Mahnung zu erzeugen. +
→ Es öffnet sich eine **PDF-Datei** mit der soeben erzeugten Mahnung.

[[tabelle-auftraege-belege-mahnung]]
.Optionen im Menü *Beleg erstellen*
[cols="1,3"]
|====
|Einstellung |Erläuterung

|**Datum**
|Das Datum der Mahnung wählen. Standardmäßig ist das aktuelle Datum voreingestellt.

|**Nummer**
|Die Nummer des Dokuments wird automatisch vergeben. Hier muss keine Angabe gemacht werden.

|**Buche Warenausgang**
|Mit **JA** oder **NEIN** auswählen, ob mit dem Erzeugen der Mahnung auch der Warenausgang gebucht werden soll.

|**Ändere Auftragsstatus auf Status**
|Einen Status wählen, in den der Auftrag nach dem Erzeugen der Mahnung wechseln soll. Wenn kein Statuswechsel gewünscht ist, die Option **Keine Änderung** wählen.

|**Varianten-ID**
|Eine **Mahngebühr** müssen Sie als eigenständigen Artikel mit einem Preis anlegen. Dieser Artikel erscheint dann als separater Posten auf der Mahnung und gibt die Mahngebühr aus. Wenn Sie den Artikel angelegt haben, können Sie die entsprechende ID der Hauptvariante hier eintragen. Diese wird dann automatisch allen generierten Mahnungen hinzugefügt. +
Mehr Informationen und Anleitungen zum Erstellen von Artikeln finden Sie auf der Handbuchseite <<artikel/artikel-verwalten#20, Artikel verwalten>>.

|**Kommentar**
|Einen Text eingeben, der auf der Mahnung unterhalb der Artikelpositionen angezeigt wird. Die Eingabe ist optional.
|====

[#300]
== Mehrere Mahnungen erzeugen

Sollen mehrere Mahnungen zugleich erzeugt werden, können Sie über einen Suchfilter im Menü **Aufträge » Fulfillment** festlegen, welche Aufträge berücksichtigt werden. Zudem können Sie in diesem Menü festlegen, ob der Status des Auftrags geändert werden soll, nachdem die Mahnungen erzeugt wurden.


[.instruction]
Mehrere Mahnungen erzeugen:

. Öffnen Sie das Menü **Aufträge » Fulfillment » Mahnungen erzeugen**.
. Nehmen Sie die Einstellungen anhand <<tabelle-einstellungen-fulfillment-mahnungen>> vor.
. Klicken Sie auf das **Zahnrad**, um die Mahnungen zu erzeugen. +
→ Es öffnet sich eine **PDF-Datei**, in der die zugehörigen Mahnungen aller von Ihnen durch die Suchfilter bestimmten Aufträge enthalten sind.

[[tabelle-einstellungen-fulfillment-mahnungen]]
.Optionen im Menü *Mahnungen erzeugen*
[cols="1,3"]
|====
|Einstellung |Erläuterung

|**Auftragsdatum**
|Begrenzen Sie die zu mahnenden Aufträge durch Auswahl eines Anfangs- und Enddatums aus den Kalendern. Es werden die Aufträge gefunden, die im gewählten Zeitraum eingegangen sind.

|**Auftragsstatus**
|Aus der Liste den Status der Aufträge wählen, die für das Erzeugen der Mahnungen berücksichtigt werden sollen.

|**Land**
|Wählen Sie ein Land aus, auf das die Suche beschränkt werden soll oder die Option **ALLE**.

|**Eigner**
|Wählen Sie einen Eigner aus, für dessen Aufträge die Mahnungen erzeugt werden sollen oder die Option **ALLE**, wenn für alle Eigner die Mahnungen erzeugt werden sollen.

|**Standort**
|Wählen Sie einen Standort aus, für den die Mahnungen erzeugt werden sollen.

|**Lager**
|Wählen Sie ein Lager oder **ALLE** aus.

|**Zahlungsart**
|Wählen Sie eine der Zahlungsarten oder *ALLE* aus.

|**Sortierung**
|Legen Sie fest, wie die erzeugten Mahnungen sortiert werden sollen. Die Optionen **Auftrags-ID** und **Kunden-ID** geben verschiedene Sortierungen aus.

|**Ändere Auftragsstatus**
|Wählen Sie einen Status, in den die Aufträge nach Erzeugung der Mahnungen wechseln sollen. +
Der Statuswechsel ist wichtig, damit für bereits bearbeitete Aufträge ein Prozess nicht noch einmal durchgeführt wird.

|**Varianten-ID**
|Eine **Mahngebühr** müssen Sie als eigenständigen Artikel mit einem Preis anlegen. Dieser Artikel erscheint dann als separater Posten auf der Mahnung und gibt die Mahngebühr aus. Wenn Sie den Artikel angelegt haben, können Sie die entsprechende ID der Hauptvariante hier eintragen. Diese wird dann automatisch allen generierten Mahnungen hinzugefügt. +
Mehr Informationen und Anleitungen zum Erstellen von Artikeln finden Sie auf der Handbuchseite <<artikel/artikel-verwalten#20, Artikel verwalten>>.

|**Kommentar**
|Einen Text eingeben, der auf allen Mahnungen unterhalb der Artikelpositionen erscheint. Die Eingabe ist optional.
|====
