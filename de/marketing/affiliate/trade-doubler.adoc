== Trade Doubler

link:http://www.tradedoubler.com/de-de/[**TradeDoubler**, window="_blank"] ist Ihr Partner für leistungsbasiertes digitales Marketing. Dank langjähriger Erfahrung, ausgefeilter Technologie und einem breiten Produktangebot bietet Ihnen das Unternehmen maßgeschneiderte Lösungen. Im Bereich **Affiliate-Marketing** verfügt Trade Doubler über langjährige Erfahrung, die Ihnen optimalen Support bei der Betreuung und Optimierung Ihrer performance-orientierten Kampagnen ermöglicht, aber auch den Schutz Ihrer Marke durch beständige Überwachung des Netzwerkes gewährleistet.

== Bei Trade Doubler registrieren

Sie benötigen einen **Account** bei Trade Doubler, um die Funktion in plentymarkets zu nutzen. **link:http://www.tradedoubler.com/pan/login?action=register[Registrieren, window="_blank"]** Sie sich bei Trade Doubler. Sie erhalten die Zugangsdaten, die Sie in plentymarkets hinterlegen müssen.

[WARNING]
.**URL-Parameter**
====
Für die Registrierung muss die **Tracking-Kennung** in der Form **tduid=XYZ**, die **Auftragsherkunft** in der Form **ReferrerID=XYZ** mit der URL **www.domain.tpld/plenty/api/affiliate.php?** gesendet werden.
====

**Beispiel**: **www.domain.tpld/plenty/api/affiliate.php?tduid=12345&amp;ReferrerID=24**

[TIP]
.Tipp
====
Hinweise zum Ausfüllen des Formblatts **Product Feed set up form**.
====

**HTTP authentication method**

Wählen Sie **None**.

**Currency**

Tragen Sie **EUR** ein.

**Encoding**

Wählen Sie **UTF-8**.

== Trade Doubler in plentymarkets einrichten

Mit den **Zugangsdaten**, die Sie von Trade Doubler erhalten haben, können Sie Trade Doubler nun in plentymarkets einrichten.

image::marketing/affiliate/assets/DE-Einstell-Mandant-Std-Affiliate-tradedoubler-01-SI.png[]

__Bild 1: **Zugangsdaten** eintragen__

[NOTE]
.Trade Doubler in plentymarkets einrichten:
====
Öffnen Sie das Menü **Trade Doubler** (Pfad siehe oben).

Tragen Sie die **Zugangsdaten** ein, wie in Tabelle 1 beschrieben.

Klicken Sie auf das **Speichern-Symbol**, um die Einstellungen zu sichern.
====

Erläuterungen zu den Optionen

[cols="a,a"]
|====
|Eintrag/Einstellung |Erläuterung

|**Organisation-ID**
|Tragen Sie die **Organisation-ID** ein, die Sie von Trade Doubler erhalten haben.

|**Event-ID**
|Tragen Sie die **Event-ID** ein, die Sie von Trade Doubler erhalten haben.

|**Secret-Code**
|Tragen Sie den **Secret Code** ein, den Sie von Trade Doubler erhalten haben.

|**Provision**
|Die Provision vereinbaren Sie mit Trade Doubler.

|**Auftragsherkunft (Referrer)**
|Wählen Sie eine Auftragsherkunft oder lassen Sie die Einstellung auf **keine Auswahl**.

|
|Sie können für Trade Doubler eine eigene Auftragsherkunft erstellen, im Menü **Einstellungen &gt; Aufträge &gt; Auftragsherkunft** und diese dann hier einstellen (siehe Bild 1). +
Dieser Auftragsherkunft können Sie eine eigene Preisspalte mit einem individuellen Preis zuordnen, wenn Sie möchten, dass dafür ein anderer Preis gelten soll.

|**Mandant (Shop)**
|Wählen Sie den **Webshop** oder **Mandant (Shop)**, dessen **Artikel-URLs** für den **Export** verwendet werden sollen, aus dem Dropdown-Menü, oder die Einstellung **Alle**, wenn alle URLs exportiert werden sollen.

|
|Für eine alternative Zuordnung des Webshops bzw. Mandant (Shop) beachten Sie die Option Artikel-Export-URL weiter unten.

|**Sprache für Artikel**
|Sprache der Artikel einstellen.

|**Bilder exportieren**
|Wählen Sie anhand der Bildposition, welche **Artikelbilder** exportiert werden sollen.

|**Artikeladresszusatz**
|Tragen Sie einen **Adresszusatz** ein, um die jeweiligen URLs entsprechend zuordnen zu können. Dieser Zusatz wird an die URL des Deep-Links der Artikel angehängt.

|
|**Beispiel**: wenn Sie den Econda Shop-Monitor verwenden, können Sie an die DeepLinks den Parameter **&amp;campaign=tradedoubler** anhängen, dadurch bekommen Sie dann bei Econda in einer Übersicht angezeigt, wie viele Besucher über diesen Affiliate-Partner gekommen sind.

|**Beschränkung**
|Wählen Sie die Option für die Auswahl der Artikel: +
**Netto-Bestand oder keine Bestandsbeschränkung** = Artikel, bei denen im Tab **Bestand** entweder **Beschränkung auf Netto-Warenbestand** oder **Keine Beschränkung** eingestellt ist. +
**Netto-Bestand** = Artikel, bei denen **Beschränkung auf Netto-Warenbestand** im Tab **Bestand** eingestellt ist. +
**Für Web-API freigegeben** = Artikel, bei denen die Option **Web-API** im Tab **Verfügbar** aktiviert ist.

|**Artikelauswahl**
|Die Standardeinstellung **1 - 4000** für die ersten 4000 Artikel wird bei mehr als 4000 Artikel um weitere Optionen erweitert. Sie können dann wählen, welcher Artikelblock ausgewählt werden soll, z.B. **4001 - 8000**, **8001 - 12000** etc. Die Artikel sind in Blocks zu je 4000 auswählbar.

|**Artikelexport-URL**
|**URL** für den **Export der Artikel** in eine XML-Datei. Wenn Sie die Zuordnung zum Webshop bzw. Multishop (siehe Option Multishop-Export) über diese URL vornehmen möchten, gehen Sie folgendermaßen vor:

|
|Fügen Sie der **Artikel-Export-URL**, die Sie dem Provider übermitteln, folgende Erweiterung an: **&amp;params$multishop=xxx** (**xxx**=**ID** des jeweiligen **Webshops**).

|
|pass:q[<u>**Shop-ID**</u>] +
**Standard Webshop** = 0 +
**Mandant (Shop)** = ID, die im Untermenü **Grundeinstellungen** des jeweiligen Mandant (Shop) angezeigt wird.

|
|Wenn Sie für die URL eine **Erweiterung** an den Provider übermitteln, ist die Einstellung unter **Mandant (Shop)-Export** (siehe oben) unwirksam.
|====
