= accountContacts
:lang: de
include::{includedir}/_header.adoc[]
:keywords: accountContacts, Report, Rohdaten
:position: 10

Der Datenformat-Creator *accountContacts* exportiert die im System vorhandenen Kontakte (Kundendaten) gemäß der im Folgenden definierten Regeln und speichert diese im Cloud-Speicher ab.

== Technische Spezifikationen

<<tabelle-spezifikationen-accountcontacts>> erläutert die verfügbaren Eigenschaften und die dazugehörigen Werte.

[[tabelle-spezifikationen-accountcontacts]]
.Technische Spezifikationen des Formats *accountContacts*
[cols="4"]
|====
|Eigenschaft |Bedeutung |Wert |Erläuterung

|dataName
|Name
|accountContacts
|Name des Creator-Prozesses und Präfix der erstellten Datei. +
Dateiname: *accountContacts-xxx.csv.gz*

|fileFormat
|Dateiformat
|CSV, GZIP
|Die Daten werden als per GZIP (RFC 1952) komprimierte CSV-Datei bereitgestellt.

|defaultActive
|Standardmäßig aktiv
|nein
|Der Export läuft erst dann automatisch im Hintergrund, wenn der Creator zuvor im Menü *Daten » Rohdaten* aktiviert wurde.

|interval
|Intervall
|täglich
|Mit dieser Einstellung werden 1 x pro Tag, gewöhnlich über Nacht, alle seit dem letzten Durchlauf geänderten Daten exportiert.

|limitPage
|Zeilen pro Anfrage
|bis zu 3.000
|Die maximale Anzahl der Datenzeilen, die in eine einzelne Datei geschrieben wird.

|limitDay
|Zeilen pro Tag
|bis zu 80.0000
|Die maximale Anzahl der Datenzeilen, die insgesamt an einem Tag geschrieben wird.

|earliestPossibleDataDate
|Frühester Zeitpunkt
|2016-01-01
|Zeitpunkt, ab dem die Rohdaten erhoben werden können. Die Daten werden nach dem Datum gefiltert, das in der Spalte *dataUpdatedAtColumnName* enthalten ist.

|dataUpdatedAtColumnName
|Spaltenbezeichnung
|ac_updated_at
|Name der Spalte, die den Zeitstempel der letzten Aktualisierung der Daten enthält.

|deleteRawDataAfterDaysMinimum
|Minimale Anzahl an Tagen bis zur Löschung
|20
|Die minimale Anzahl an Tagen, bis die generierte Rohdaten-Datei wieder vom Cloud-Speicher entfernt wird. Eine Weiterverarbeitung muss somit vorher erfolgen. Die genaue Dauer kann im Menü *Daten » Rohdaten* definiert werden.

|deleteRawDataAfterDaysMaximum
|Maximale Anzahl an Tagen bis zur Löschung
|80
|Die maximal Anzahl an Tagen, bis die generierte Rohdaten-Datei wieder vom Cloud-Speicher entfernt wird. Eine Weiterverarbeitung muss somit vorher erfolgen. Die genaue Dauer kann im Menü *Daten » Rohdaten* definiert werden.

|====

== Datenspalten

<<tabelle-datenspalten-accountcontacts>> erläutert die Datenspalten des Formats *accountContacts*, die beim Export der Rohdaten in der CSV-Datei gespeichert werden.

[[tabelle-datenspalten-accountcontacts]]
.Datenspalten des Formats *accountContacts*
[cols="1,3,3"]
|====
|Spaltenname |Vollständige Bezeichnung |Erläuterung

|plenty_id
|plenty id
|ID des plentymarkets Systems

|ac_id
|account contact id
|ID des Kontakts

|a_id
|account id
|ID der Firma, sofern vorhanden

|act_id
|account contact type id
|ID des Kontakttyps +
Mögliche Werte: +
1 = Kunde +
2 = Interessent +
3 = Handelsvertreter +
4 = Lieferant +
5 = Hersteller +
6 = Partner

|acc_id
|account contact class id
|ID der Kontaktklasse

|r_id
|referrer id
|ID der Auftragsherkunft, über die der Datensatz angelegt wurde

|u_id
|user id
|ID des Benutzers, der als Eigner des Kontakts hinterlegt wurde

|ac_name
|account contact name
|Vorname und Nachname des Kontakts

|ac_country
|account contact country
|Land des Kontakts als ISO2-Ländercode

|ac_lang
|account contact lang
|Sprache des Kontakts, die im Kontaktdatensatz gespeichert ist. +

|ac_rating
|account contact rating
|Bewertung des Kontakts +
Dient der internen Bewertung des Kontakts. 5 graue Sterne stehen für keine Bewertung. 5 gelbe Sterne stehen für die beste, 5 rote Sterne für die schlechteste Bewertung: +
0 = 5 graue Sterne (keine Bewertung) +
5 = 5 gelbe Sterne (beste Bewertung) +
4 = 4 gelbe Sterne +
3 = 3 gelbe Sterne +
2 = 2 gelbe Sterne +
1 = 1 gelber Stern +
-1 = 1 roter Stern +
-2 = 2 rote Sterne +
-3 = 3 rote Sterne +
-4 = 4 rote Sterne +
-5 = 5 rote Sterne (schlechteste Bewertung)

|ac_updated_at
|account contact updated at
|Zeitstempel der letzten Aktualisierung +
Datumsformat: MM/DD/YYYY HH:MI:SS TZ

|ac_created_at
|account contact created at
|Zeitstempel der Erstellung +
Datumsformat: MM/DD/YYYY HH:MI:SS TZ

|====
