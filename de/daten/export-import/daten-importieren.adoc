Diese Handbuchseite beschreibt verschiedene Möglichkeiten des manuellen Datenimports über den dynamischen Import. Ein Import von Daten ist ratsam, wenn du z.B. neue Artikel, Kundendaten oder Lagerbestände in dein plentymarkets System einpflegen möchtest und die Daten bereits digital vorliegen.

====

== Datenformat erstellen

Das Datenformat, das du importierst, muss der Konfiguration des Datenformats in plentymarkets entsprechen, damit die Daten zugeordnet werden. Hierzu gehört auch, dass die Schreibweise der Datenfelder der Schreibweise in plentymarkets entspricht. Auf der Handbuchseite eines <<daten/export-import/datenformate#, Datenformats>> findest du die Namen der Datenfelder aufgelistet. Beachte auch die Hinweise zur Angabe von Pflichtfeldern und Abgleichfeldern. Du hast die Möglichkeit, mit Hilfe eines Tabellenkalkulationsprogramms selbst eine Datei für den Import zu erstellen.

Du kannst dir diese Arbeit erheblich erleichtern, indem du das Datenformat vorab in eine Datei exportierst, deine Daten in der Datei bearbeitest und diese dann wieder importierst. Diese Vorgehensweise ist nur bei Datenformaten möglich, die sowohl einen Import als auch einen Export erlauben. Weitere Informationen zum Umgang mit Export- und Importdateien findest du im Kapitel <<daten/export-import/daten-exportieren#05, Datei für den Datenaustausch anpassen>> auf der Handbuchseite <<daten/export-import/daten-exportieren#, Daten exportieren>>. Gehe wie im Folgenden beschrieben vor, um deine Daten in eine Datei zu exportieren:

[.instruction]
Datenformat erstellen:

. Öffne das Menü *Daten » Dynamischer Export*.
. Wechsele in das Tab *Neues Datenformat*.
. Wähle einen *Datenformat-Typ* aus der Dropdown-Liste.
. Ändere ggf. den automatisch vergebenen *Datenformatsnamen*.
. Aktiviere die Datenfelder, die in der Exportdatei vorhanden sein sollen.
. Wähle eine *Importaktion* pro Datenfeld. +
→ Beachte die Hinweise zu Abgleichfeldern auf den Unterseiten der Handbuchseite <<daten/export-import/datenformate#, Datenformate>>.
. Wähle ggf. *Filter* oder *Aktionen*.
. *Speichere* (icon:save[role="green"]) das Datenformat. +
→ Die Reihenfolge der Datenfelder wird anhand der Werte in der Spalte *Position* gespeichert.

[#25]
[TIP]
.Abgleich und Import
====
Bei einem Datenformat stehen für die Datenfelder die Importaktionen *Abgleich* und *Import* zur Verfügung. +
Mit der Einstellung *Import* werden die Daten importiert - unabhängig davon, ob bereits Daten vorhanden sind oder nicht. Das bedeutet, dass die Daten bei einem Import eventuell überschrieben werden. Mit der Einstellung *Abgleich* wird das Datenfeld vor einem Import mit bereits im System vorhandenen Daten abgeglichen. Die Daten werden nur importiert, wenn noch keine Daten vorhanden sind.
====

[#20]
== Dynamischen Import nutzen

Der dynamische Datenimport erfolgt im Menü *Daten » Dynamischer Import*. Du kannst hier selbst erstellte Dateien oder die Datenformate, die du im Menü *Daten » Dynamischer Export* erstellt hast (sog. Kundenformate), wieder importieren. Im Menü *Daten » Dynamischer Import* stehen dir neben den Kundenformaten in der Dropdown-Liste *Format* zwei Spezialformate zur Verfügung:

* das Spezialformat *ItemParcelServicePreset*, mit dem du Versandprofile importierst
* das Spezialformat *IncomingItems*, mit dem du Wareneingangsdaten importierst

Die Spezialformate sind reine Importformate und stehen daher nur im Menü *Daten » Dynamischer Import* zur Verfügung. Der Import der Spezialformate funktioniert auf die gleiche Weise, wie du auch die anderen Datenformate importierst. Die Vorgehensweise wird im Folgenden beschrieben.

[NOTE]
.Spaltenkopfbezeichnungen prüfen und anpassen
====
Wenn Daten aus einem externen System importiert werden sollen oder du die Datei selbst erstellt hast, müssen die Spaltenkopfbezeichnungen in der Datei mit der von plentymarkets vorgegebenen Schreibweise der Datenfelder (Feldnamen) exakt übereinstimmen. +
Kann plentymarkets beim Import keine Zuordnung zu einer gewählten Exportvariante herstellen, wird der Import mit einer Fehlermeldung abgebrochen. Die Fehlermeldung findest du im Bereich *Bearbeitungsstatus*. Prüfe die Spaltenkopfbezeichnungen der zu importierenden Datei und passe diese bei Bedarf an.
====

[.instruction]
Daten per dynamischem Import importieren:

. Öffne das Menü *Daten » Dynamischer Import*.
. Nimm die Einstellungen vor. Beachte dazu die Erläuterungen in <<tabelle-einstellungen-menü-daten-dynamischer-import>>. +
*_Hinweis:_* Pro Import können maximal 6000 Datensätze importiert werden.
. Klicke auf *Import*.
. Prüfe, ob die importierten Daten im System gespeichert wurden.

[IMPORTANT]
.Bearbeitungsstatus
====
Nachdem du die Datei importiert hast, erhältst du eine Meldung im Bereich *Bearbeitungsstatus*. Wenn Datenzeilen erfolgreich importiert wurden, wird die Anzahl der importierten Zeilen angegeben. Im Fehlerfall steht dort der Eintrag *0* und eine Fehlermeldung wird angezeigt. Prüfe das Feldtrennzeichen oder die Codierung der Datei. Die Datei sollte immer UTF-8-codiert sein.
====

Beachte, dass die Einstellungen, die angezeigt werden, von dem Datenformat abhängig sind, das du für den Import gewählt hast. Für das Datenformat *Stock* z.B. wählst du statt einer Abgleichoption das *Lager* und den *Barcode*. Die Einstellungen, die in <<tabelle-einstellungen-menue-daten-dynamischer-import>> beschrieben sind, gelten für die Mehrheit der Datenformate.

[[tabelle-einstellungen-menue-daten-dynamischer-import]]
.Einstellungen im Menü *Daten » Dynamischer Import*
[cols="1,3"]
|====
|Einstellung |Erläuterung

|*Feldtrennzeichen*
|Feldtrennzeichen wählen, das in der Importdatei eingestellt ist.

|*Format*
|Datenformat wählen, das du für den Datenimport vorbereitet hast. +
*_Hinweis:_* Je nachdem, welches Datenformat du hier wählst, ändern sich die folgenden Einstellungen.

|*Wenn beim Abgleich keine Übereinstimmung*
|Eine der folgenden Optionen wählen:  +

|*Datei*
|Importdatei vom Speicherort wählen.
|====

== eBay-Listings importieren

Im Menü *Daten » eBay-Listings* importierst du eBay-Listings. Es wird sowohl ein Artikelstammdatensatz als auch ein Listing in plentymarkets erstellt. Im Bereich *Bearbeitungsstatus* hast du während des Importprozesses Einsicht in den aktuellen Verlauf des Imports. Wie du die Listings importierst, ist auf der Handbuchseite von <<maerkte/ebay/ebay-einrichten#, eBay>> im Kapitel <<maerkte/ebay/ebay-einrichten#11000, eBay-Listings importieren>> beschrieben.

== Anwendungsbeispiel: Artikel erstellen und Artikeldaten anpassen

Im Menü *Daten* erstellst du Artikel oder änderst bestehende Artikeldaten. Diese Funktion bietet sich an, wenn du z.B. Artikellisten von deinem Lieferanten in plentymarkets importieren möchtest oder den Preis deiner Artikel anpassen möchtest. Erstelle zunächst das Datenformat *Item* im Menü *Daten » Dynamischer Export*. Exportiere dann die Daten in eine Datei und bearbeite die Daten in der Datei. Abschließend importierst du diese Datei im Menü *Daten » Dynamischer Import*.

[#120]
== Anwendungsbeispiel: Warenbestand importieren

Du hast auch die Möglichkeit, den kompletten Warenbestand zu importieren, bei dem aktuelle physische Warenbestände in deinem plentymarkets System überschrieben werden. Nutze dazu das Datenformat <<daten/export-import/datenformate/stock#, Stock>>.

[WARNING]
.Warenbestand wird gelöscht
====
Bei diesem Vorgang werden die aktuellen physischen Warenbestände durch den Import ersetzt, d.h. dein gesamter Warenbestand wird gelöscht. Reservierte Warenbestände bleiben dabei erhalten.

Wenn du nicht den kompletten Warenbestand importieren möchtest, sondern nur Wareneingänge per Datenimport hinzufügen möchtest, verwende das Spezialformat *IncomingItems*.
====

Wenn du die Importdatei vor dem Import bearbeiten möchtest, beachte die folgenden Erläuterungen zu den Datenfeldern. <<tabelle-datenfelder-datenformat-stock>> erläutert eine Auswahl der in der exportierten Datei enthaltenen Datenfelder.
