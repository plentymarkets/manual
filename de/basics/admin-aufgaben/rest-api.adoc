= REST-API-Schnittstellen einrichten
:lang: de
include::{includedir}/_header.adoc[]
:keywords:
:description: Nutzen Sie REST-API, um mit externen Systemen zu kommunizieren.
:position: 40

Bei dem Begriff REST-API handelt es sich um eine Zusammensetzung aus zwei Akronymen. Dabei steht REST für _Representational State Transfer_ (dt. etwa: repräsentative Zustandsübertragung) und API für _application programming interface_, eine Schnittstelle zwischen zwei Systemen. Allgemein gesagt ermöglicht diese Schnittstelle externen Aufrufen, Daten an das plentymarkets System zu senden oder Daten von dem System abzurufen. Beispielsweise könnten dadurch also alle Kontakte aus der Kontaktlisten auf einmal abgerufen werden.

Auch in manchen Plugins, etwa link:https://marketplace.plentymarkets.com/plugins/channels/marktplaetze/ebayoauth2_4787[eBay OAuth2^] und link:https://marketplace.plentymarkets.com/plugins/channels/marktplaetze/etsy_4689[Etsy^], werden sogenannte REST-Routen verwendet, um mit dem plentymarkets System zu kommunizieren. Um dies zu ermöglichen, müssen bestimmte Berechtigungen vergeben werden.

[#endanwender]
== Für Endanwender

Sollen Plugins mit REST-Routen im System eingebunden werden, muss diesen Plugins über einen Benutzer der Benutzerklasse Variabel ermöglicht werden, auf Teile des plentymarkets Systems zuzugreifen.

[.instruction]
Benutzerkonto der Klasse Variable erstellen:

. Öffnen Sie das Menü *System » Systemeinstellungen » Einstellungen » Benutzer » Konten*.
. Klicken Sie auf *Neu*. +
→ Das Fenster *Neuen Benutzer anlegen* wird geöffnet.
. Geben Sie den *Benutzernamen* ein.
. Geben Sie den *realen Namen* ein.
. Geben Sie das *Passwort* ein.
. Wiederholen Sie das Passwort.
. Wählen Sie die Benutzerklasse *Variabel*.
. *Speichern* Sie die Einstellungen.

Der Benutzername darf kein Leerzeichen enthalten und muss mindestens 5 Zeichen lang sein.

Im Tab REST API legen Sie fest, welche Berechtigungen ein Benutzerkonto der Klasse Variabel für die REST API erhält. Standardmäßig sind keine REST-API-Berechtigungen vergeben. Berechtigungen werden für die verschiedenen Module und Submodule der REST API separat vergeben. Unter Autorisierung legen Sie zum Beispiel fest, ob REST-API-Benutzerkonten Berechtigungen anzeigen bzw. Benutzerrollen anzeigen, erstellen und löschen dürfen.

Welche Rechte ein Plugin benötigt, ist in der Beschreibung des jeweiligen Plugins vermerkt.

[.instruction]
REST-API-Bereiche freischalten:

. Öffnen Sie das Menü *System » Systemeinstellungen » Einstellungen » Benutzer » Konten*.
. Öffnen Sie einen Benutzer.
. Öffnen Sie das Tab *Berechtigung*.
. Öffnen Sie das Tab *REST API*.
. Aktivieren Sie komplette Bereiche oder klappen Sie Bereiche auf, um einzelne Berechtigungen zu aktivieren, die dem Benutzerkonto zur Verfügung stehen sollen.
. *Speichern* Sie die Einstellungen.

Nachträglich ergänzte, untergeordnete Berechtigungen eines Hauptmoduls werden automatisch für einen Benutzerkonto aktiviert, wenn das Hauptmodul aktiviert ist. Sie können diese Funktion deaktivieren und neue, untergeordnete Berechtigungen manuell aktivieren.

[.instruction]
Neue, untergeordnete Berechtigungen manuell aktivieren:

. Öffnen Sie das Menü *System » Systemeinstellungen » Einstellungen » Benutzer » Konten*.
. Öffnen Sie einen Benutzer.
. Öffnen Sie das Tab *Berechtigung*.
. Öffnen Sie das Tab *REST API*.
. Klicken Sie auf *Neue, untergeordnete Berechtigungen manuell aktivieren*.
. *Speichern* Sie die Einstellungen.
