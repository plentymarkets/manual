= plentyBase installieren
:lang: de
include::{includedir}/_header.adoc[]

:description: plentyBase installieren

Für die folgenden Abläufe in plentymarkets muss das Tool plentyBase auf dem Rechner installiert und gestartet sein:

* Drucker konfigurieren
* Dokumente speichern
* Dokumente drucken
* Druckverlauf aufrufen
* Prozesse im Browser erstellen
* Prozesse im Browser ausführen

Die Installationsdateien für plentyBase finden Sie im **link:https://marketplace.plentymarkets.com/plugins/integration/plentybase_5053[plentyMarketplace]**.

Wenn plentyBase auf dem Rechner installiert ist, können diese Abläufe über die Terra-Version des plentymarkets Backend ausgeführt werden. Alternativ zu plentyBase steht der <<basics/automatisierung/plentymarkets-client#, **plentymarkets Client**>> zur Verfügung.

[#30]
== plentyBase installieren

plentyBase installieren Sie wie jedes andere Programm auf dem Rechner. Da der Installationsvorgang je nach Betriebssystem abweicht, werden hier nur für plentyBase spezifische Einstellungen beschrieben.

. Führen Sie die Installationsdatei aus.
. Wählen Sie im Installationsprozess den Port, über den der Rechner mit plentyBase kommunizieren soll. +
**__Hinweis:__** Standardmäßig ist der Port 7331 angegeben, der auf den meisten Rechnern frei ist. Ändern Sie den Standardwert nur, wenn der Port 7331 auf dem Rechner belegt ist. +
**__Tipp:__** Mit den folgenden Befehlen prüfen Sie über die Befehlszeile, welche Ports des Rechners belegt sind: +
* Windows: __netstat -ano \| find /i „Abhören“__ +
* Mac OSX: __sudo lsof -i -P \| grep -i LISTEN__
. Schließen Sie die Installation ab. +
→ plentyBase wird auf dem Rechner installiert. +
→ plentyBase wird zu den Autostart-Programmen des Betriebssystems hinzugefügt.
. Klicken Sie auf das plentyBase Icon in der Taskleiste des Rechners.
. Wählen Sie **Konfiguration öffnen** aus der Dropdown-Liste. +
→ Das Fenster **Konfiguration** wird geöffnet. +
. Aktivieren Sie die Option **Beim Start von plentyBase auf Updates prüfen**. +
. Klicken Sie auf *OK*, um die Einstellungen zu speichern. +


[#40]
== plentyBase Verbindungsdaten in plentymarkets speichern

Nach der Installation speichern Sie die Verbindungsdaten in plentymarkets, damit plentymarkets mit plentyBase kommunizieren kann.

[.instruction]
plentyBase Verbindungsdaten in plentymarkets speichern:

. Loggen Sie sich in die Terra-Version des plentymarkets Systems ein.
. Klicken Sie rechts oben auf das Benutzersymbol.
. Wählen Sie **plentyBase** aus der Dropdown-Liste. +
→ Das plentyBase Dashboard wird geöffnet.
. Wechseln Sie in die Ansicht **Einstellungen**.
. Nehmen Sie die Einstellungen vor. Beachten Sie die Erläuterungen in Tabelle 1. +
*_Tipp:_* Die Einstellungen werden automatisch gespeichert. +
→ plentyBase ist auf dem Rechner aktiviert und stellt die Funktionalität für alle plentymarkets Benutzer bereit. +
→ In der Taskleiste des Rechners wird das plentyBase Icon angezeigt, das zeigt, dass das Tool ausgeführt wird. +
. Kopieren Sie den Access Token in die Zwischenablage. +
. Klicken Sie auf das plentyBase Icon in der Taskleiste des Rechners.
. Wählen Sie **Konfiguration öffnen** aus der Dropdown-Liste. +
. Aktivieren Sie die Option **Authentifizierung benutzen**. +
. Klicken Sie auf **Access Token verwalten**. +
→ Das Fenster **Access Token verwalten** wird geöffnet. +
. Klicken Sie auf **Hinzufügen**. +
→ Das Fenster **Access Token hinzufügen** wird geöffnet. +
. Kopieren Sie den Access Token in das Feld **Access Token**. +
. Geben Sie optional eine Beschreibung des Tokens ein. +
. Klicken Sie auf **Hinzufügen**. +
→ Das Access Token wird gespeichert.

.plentyBase Verbindungsdaten speichern
[cols="1,2"]
|====
|Einstellung |Erläuterung

|**Server-Adresse**
|Die gespeicherte Adresse \https://local.plentybase.de nicht ändern.

|**Port**
|Der Port, der plentyBase bei der Installation zugeordnet wurde. Standardwert ist 7331. Wenn bei der Installation ein anderer Port angegeben wurde, die Nummer dieses Ports eingeben. +
**__Hinweis:__** Die Ports 0 bis 1023 sind System-Ports und können nicht verwendet werden.

|**Verbindung testen**
|Prüft, ob eine Verbindung von plentymarkets zu plentyBase besteht.

|**Token generieren**
|Das API Access Token dient zur Authentifizierung bei der plentyBase API-Schnittstelle. Auf **Token generieren** klicken und die Generierung bestätigen, um ein neues Access Token zu generieren. +
*_Hinweis:_* Durch Generieren eines neuen Access Tokens wird das alte Token überschrieben. Neues Token nach der Generierung in den Einstellungen des plentyBase Tools speichern.

|**Einstellungen zurücksetzen**
|Setzt Server-Adresse und Port auf die Standardwerte zurück.
|====

[#50]
== plentyBase in einem Netzwerk zugänglich machen

plentyBase kann auf einem Rechner innerhalb eines Netzwerks installiert und für die anderen Rechner im Netzwerk zugänglich gemacht werden. Um plentyBase innerhalb eines Netzwerks zu nutzen, müssen die IP-Adresse und der Host-Name des Rechners, auf dem plentyBase installiert ist, in die HOSTS-Datei der anderen Rechner eingetragen werden, die auf das Netzwerk zugreifen.

[#60]
=== plentyBase auf Netzwerkrechner installieren

* <<basics/automatisierung/plentybase-installieren#30, **Installieren Sie plentyBase**>> auf einem Rechner des Netzwerks.
* Notieren Sie sich die IP-Adresse des Host-Rechners, auf dem plentyBase installiert wurde. +
**__Hinweis:__** Es wird empfohlen, dem Host-Rechner eine statische IP-Adresse zuzuweisen.
* Wenn Sie nicht den Standard-Port 7331 verwendet haben, notieren Sie sich außerdem den Port, den Sie bei der Installation angegeben haben.

[#70]
=== IP-Daten in HOSTS-Datei der anderen Rechner im Netzwerk speichern

Die HOSTS-Datei ist eine lokale Textdatei des Betriebssystems, in der Hostnamen bestimmten IP-Adressen zugeordnet werden. In die HOSTS-Datei des Client-Rechners tragen Sie die IP-Adresse und den Host-Namen des Rechners im Netzwerk ein, auf dem plentyBase installiert ist. Nur so kann der Client-Rechner auf plentyBase zugreifen. Zur Bearbeitung der HOSTS-Datei benötigen Sie Administratorrechte auf dem Client-Rechner.

[WARNING]
.Bearbeiten der HOSTS-Datei auf eigene Gefahr
====
Die HOSTS-Datei ist eine wichtige Systemdatei. Die Bearbeitung dieser Datei erfolgt auf eigene Gefahr. plentymarkets übernimmt keine Gewährleistung oder Haftung für etwaige Schäden, die durch die Bearbeitung der HOSTS-Datei direkt oder mittelbar entstehen können. Beachten Sie auch folgende Hinweise:

* Erstellen Sie eine Sicherungskopie der Hosts-Datei, bevor Sie die Datei bearbeiten.
* Löschen oder bearbeiten Sie bereits in der Datei vorhandenen Einträge nicht.
====

[#80]
==== HOSTS-Datei in Windows bearbeiten

Gehen Sie wie unten beschrieben vor, um die IP-Adresse und den Host-Namen des Rechners, auf dem plentyBase installiert ist, in die HOSTS-Datei eines Windows-Rechners einzutragen. Sie benötigen die IP-Adresse des Rechners, auf dem plentyBase installiert ist. Erstellen Sie eine Sicherungskopie der Hosts-Datei, bevor Sie die Datei bearbeiten.

[.instruction]
HOSTS-Datei in Windows bearbeiten:

. Klicken Sie links unten auf das Windows-Symbol.
. Nur Windows 8: Navigieren Sie zur Kacheloberfläche.
. Geben Sie **Editor** in die Suche ein. +
→ Der Editor wird in der Liste der Programme aufgeführt.
. Klicken Sie mit der rechten Maustaste auf die Editor-Verknüpfung und wählen Sie **Als Administrator ausführen**.
. Bestätigen Sie die Meldung mit **Ja**. +
→ Der Editor wird geöffnet.
. Klicken Sie auf **Datei » Öffnen**.
. Navigieren Sie zum Pfad C:\Windows\System32\Drivers\etc. +
__**Tipp:**__ Wählen Sie im Dropdown-Menü rechts unten **Alle Dateien**. +
→ Die Hosts-Datei wird geöffnet und kann bearbeitet werden.
. Fügen Sie am Ende der Datei die IP-Adresse des Rechners hinzu, auf dem plentyBase installiert ist, gefolgt von der Host-Adresse local.plentybase.de. Daraus ergibt sich das folgende Muster: +
**[IP-Adresse des Host-Rechners] local.plentybase.de** +
**__Beispiel:__** 123.123.1.123 local.plentybase.de +
**__Hinweis:__** Mit # beginnende Zeilen sind Kommentare und werden nicht ausgelesen. Ihr Eintrag darf daher nicht mit # beginnen.
. **Speichern** Sie die Datei.
. __**Tipp:**__ Wenn auf dem Rechner plentymarkets im Browser geöffnet ist, laden Sie den Browser-Tab neu.
. Wenn Sie bei der Installation auf dem Host-Rechner einen anderen Port als den Standard-Port gewählt haben, geben Sie <<basics/automatisierung/plentybase-installieren#40, **den Port des Host-Rechners**>> im plentymarkets Backend ein.

[#90]
==== HOSTS-Datei in Mac OSX bearbeiten

Gehen Sie wie unten beschrieben vor, um die IP-Adresse und den Host-Namen des Rechners, auf dem plentyBase installiert ist, in die HOSTS-Datei eines Mac OS-Rechners einzutragen. Sie benötigen die IP-Adresse des Rechners, auf dem plentyBase installiert ist. Erstellen Sie eine Sicherungskopie der Hosts-Datei, bevor Sie die Datei bearbeiten.

[.instruction]
HOSTS-Datei in Mac OSX bearbeiten:

. Öffnen Sie den Finder.
. Klicken Sie in der Task-Leiste auf **Gehe zu &gt; Gehe zum Ordner ...**. +
→ Das Fenster **Den folgenden Order öffnen:** wird angezeigt.
. Geben Sie **/private/etc** in das Feld ein. +
→ Der Ordner etc wird geöffnet.
. Kopieren Sie die Datei **hosts**.
. Legen Sie die Kopie in einem anderen Ordner oder auf dem Desktop ab.
. Öffnen Sie die Kopie der Datei **hosts** mit TextEdit oder einem anderen Editor.
. Fügen Sie am Ende der Datei die IP-Adresse des Rechners hinzu, auf dem plentyBase installiert ist, gefolgt von der Host-Adresse local.plentybase.de. Daraus ergibt sich das folgende Muster: +
**[IP-Adresse des Host-Rechners] local.plentybase.de** +
**__Beispiel:__** 123.123.1.123 local.plentybase.de +
**__Hinweis:__** Mit # beginnende Zeilen sind Kommentare und werden nicht ausgelesen. Ihr Eintrag darf daher nicht mit # beginnen.
. **Speichern** Sie die Datei.
. Ziehen Sie die bearbeitete Kopie der Datei per Drag-and-Drop in den Ursprungsordner **/private/etc**. +
→ Eine Warnung wird angezeigt.
. Bestätigen Sie, dass Sie die nicht bearbeitete Datei ersetzen möchten.
. Geben Sie Ihr Benutzerkennwort ein, um den Ersetzungsvorgang zu bestätigen. +
→ Die Datei wird gespeichert.
. __**Tipp:**__ Wenn auf dem Rechner plentymarkets im Browser geöffnet ist, laden Sie den Browser-Tab neu.
. Wenn Sie bei der Installation auf dem Host-Rechner einen anderen Port als den Standard-Port gewählt haben, geben Sie <<basics/automatisierung/plentybase-installieren#40, **den Port des Host-Rechners**>> im plentymarkets Backend ein.
