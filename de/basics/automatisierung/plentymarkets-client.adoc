== plentymarkets Client

Der **plentymarkets Client** ermöglicht die optimale Verknüpfung von plentymarkets mit lokal installierten Endgeräten. So können Sie z.B. <<basics/arbeiten-mit-plentymarkets/drucker#, **Drucker**  >> direkt mit dem System verbinden und haben somit über nur eine Software Zugriff auf alle Funktionen und müssen nicht zwischen unterschiedlichen Programmen wechseln. Der Client ermöglicht darüber hinaus die Konfiguration von <<basics/automatisierung/prozesse#, **Prozessen**  >>, mit denen Sie detaillierte Abläufe Ihres Unternehmens automatisieren.

== Download

Öffnen Sie die Webseite zum link:https://www.plentymarkets.eu/software/downloads/plentymarkets-client/[**plentymarkets Client**].

Verwenden Sie dort den aktuellsten Download-Link und starten Sie den Download durch einen Klick auf den Link Ihres Betriebssystems, d.h. Windows oder Mac OS. Installieren Sie den Client und folgen Sie dabei den Hinweisen der Installationsroutine.

== Client installieren

Der **plentymarkets Client** wird nicht in das plentymarkets-System integriert, sondern als Software auf Ihrem Rechner installiert.

=== Systemvoraussetzungen

Folgende Voraussetzungen gelten für die Installation und den Betrieb unter Windows:

* Internetverbindung
* Ein plentymarkets-System der Version 5.000 oder höher
* Windows 7 oder Windows 8, 32- oder 64-Bit
* **Wichtig**: Java VM muss als 32-Bit-Version installiert sein

[TIP]
.Betriebssystem-Info
====
Informationen zum Betriebssystem finden Sie bei Windows 7 z.B. im Menü **Systemsteuerung » System und Sicherheit » System**.
====

Führen Sie die Installation als Administrator aus. Sie erhalten die Option Als Administrator ausführen, indem Sie einen Klick mit der rechen Maustaste auf die exe-Datei.

Folgende __**Voraussetzungen**__ gelten für die Installation und den Betrieb unter __**Mac OS**__:

* Internetverbindung
* Ein plentymarkets-System der Version 5.000 oder höher
* Mac OS X 10.9 und höhere Versionen
* **Wichtig**: Java VM muss als 32-Bit-Version installiert sein

=== Hinweise zur Installation unter Windows (32 Bit bzw. ältere Systeme)

Für die Installation unter Windows werden folgende Erweiterungen (Packages) benötigt:

* **link:http://www.microsoft.com/de-de/download/details.aspx?id=30653[Microsoft.Net Framework 4.5, window="_blank"]**
* Microsoft Visual C++ 2010 Redistributable Package **link:http://www.microsoft.com/de-de/download/details.aspx?id=5555[32-Bit-Version, window="_blank"]**
* Microsoft Visual C++ 2010 Redistributable Package **link:http://www.microsoft.com/de-de/download/details.aspx?id=14632[64-Bit-Version, window="_blank"]**

Prüfen Sie in Ihrem Windows-Betriebssystem im Menü **Systemsteuerung » Programme » Programme und Funktionen** (Windows 7), ob diese Erweiterungen installiert sind.

image::basics/automatisierung/assets/DE-Tools-Client-01.png[]

__Bild 1: Microsoft-Erweiterungen im Menü Programme prüfen__

Wenn diese Erweiterungen in Ihrem System nicht vorhanden sind, installieren Sie diese, bevor Sie den plentymarkets Client installieren. Über die obigen Links finden Sie weitere Informationen zu den Erweiterungen sowie den Download.

=== Hinweis zur Programmverknüpfung unter Windows

Bei Installation unter Windows kann es vorkommen, dass die automatisch angelegte Verknüpfung nicht funktioniert und der Client somit nicht gestartet wird. Wenn dies der Fall ist, muss die Verknüpfung gelöscht werden.

image::basics/automatisierung/assets/DE-Tools-Client-02.png[]

__Bild 2: Verknüpfung entfernen__

Löschen Sie die Verknüpfung (Bild 2) und erstellen Sie eine neue Verknüpfung.

[.instruction]
Neue Verknüpfung erstellen:

. Öffnen Sie das Menü, in der sich die Anwendungsdatei zum Start des Clients befindet **C:/Programme/plentymarkets**.
. Klicken Sie mit der rechten Maustaste auf die Anwendungsdatei und wählen Sie **Verknüpfung erstellen** aus dem Kontextmenü. +
→ Es wird eine Verknüpfung auf dem Desktop Ihres Rechners erstellt.
. Ziehen Sie die Verknüpfung ggf. auf den **Windows-Startknopf**, damit Sie den Client wieder über **Start » plentymarkets** starten können.

== Client mit plentymarkets verbinden

Nachdem der Client erfolgreich installiert und gestartet wurde, verbinden Sie den Client mit dem gewünschten plentymarkets-System.

image::basics/automatisierung/assets/DE-Tools-Client-03.png[]

__Bild 3: Client verbinden__

[.instruction]
Client mit plentymarkets verbinden:

. Klicken Sie im Login-Fenster in das Dropdown-Menü **URL**.
. Wählen Sie dort das gewünschte System oder klicken Sie auf den Eintrag **neue URL eingeben** und tragen Sie die **URL** dann in die Zeile ein.
. Klicken Sie auf **Speichern**.
. Tragen Sie den **Benutzernamen** und das **Passwort** zum System ein.
. Klicken Sie auf **Login**.

[TIP]
.URL
====
Geben Sie eine verschlüsselte URL (https) ein, damit der Client verschlüsselt kommuniziert. Geben Sie eine unverschlüsselte URL (http) ein, damit der Client unverschlüsselt kommuniziert.

Der Client unterstützt nur verbundene Netzlaufwerke, keine UNC-Pfade.
====

== Update

Wir entwickeln den plentymarkets Client kontinuierlich weiter und stellen Ihnen diese Updates automatisch zur Verfügung. Wenn Sie den Client starten, erfolgt eine automatische Prüfung auf Updates. Der Update-Vorgang erfolgt menügeführt und ist selbsterklärend.

[IMPORTANT]
.Update nicht unterbrechen
====
Achten Sie darauf, dass die Verbindung zum plentymarkets-System während des Updates nicht getrennt wird, da das Update sonst nicht korrekt installiert werden kann.
====

== Cache löschen

Im Cache werden bestimmte Daten zwischengespeichert, um Inhalte schneller anzuzeigen. Teilweise sehen Sie dann aber nicht die aktuellen Inhalte. Wenn Sie sichergehen möchten, dass Ihnen die aktuellsten Inhalte angezeigt werden, löschen Sie den Cache.

image::basics/automatisierung/assets/DE-Tools-Client-04.png[]

__Bild 4: Cache löschen im Menü Extras__

[.instruction]
Cache löschen

. Öffnen Sie das Menü **plentymarkets Client » Extras » Clear cache**.
. Klicken Sie auf **Clear cache** +
→ Der Cache wird gelöscht.

Falls Sie einmal Probleme beim Login in den Client haben, löschen Sie ebenfalls zunächst den Cache.
