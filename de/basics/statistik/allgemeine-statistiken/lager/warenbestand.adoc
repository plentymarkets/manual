== Warenbestand

Zeigt den __**aktuellen Warenbestand**__ nach __**Varianten**__ gesplittet an. Es wird jeweils der __**Warenwert**__ und die __**Artikelanzahl**__ des __**verfügbaren**__ und des __**reservierten Bestandes**__ sowie viele weitere Parameter zum jeweiligen Bestand angezeigt.

== Statistik-Details

Entsprechend des gewählten __**Lagers**__ wird der __**Warenwert pro Variante**__ und der __**Warenbestand**__ angezeigt. Sind keine Warenbestände für eine Variante vorhanden, werden keine Daten angezeigt. Sie können mit dieser Statistik z.B. erkennen, welche verfügbaren und reservierten Warenwerte und Warenbestände im jeweiligen Lager oder insgesamt zu welchen Varianten vorhanden sind.

In Bild 1 wird z.B. für die oberste Variante ein __**Verfügbarer Warenwert**__ von __**700.000,00 Euro**__ angezeigt sowie ein __**Physischer Warenwert**__ von __**800.000,00 Euro**__. Als __**reservierter Warenbestand**__ wird __**1 Artikel**__ angezeigt.

[NOTE]

== Einstellungen

Legen Sie zunächst wie auf der Handbuchseite **<<basics/statistik#, Statistik>>** beschrieben eine neue Statistik an. Für die Statistik **Warenbestand** stehen dann die in Tabelle 1 beschriebenen __**Einstellungen**__ zur Verfügung.

image::basics/statistik/allgemeine-statistiken/lager/assets/DE-Statistik-Warenbestand-02-SI.png[]

__Bild 2: Statistik **Warenbestand**; **Einstellungen**__

Die folgende Tabelle erläutert die Einstellungen im Menü **Start » Statistik → Statistik-Typ: Lager / Warenbestand**:

[cols="a,a"]
|====
|Einstellung |Erläuterung

|
|Diagramm-Anzeige

|**Diagrammtyp**
|**Daten-Tabelle** = Datenübersicht in Tabellenform

|**Export**
|Die Daten des Diagramms werden in eine __**CSV-Datei**__ exportiert. Dabei werden die Parameter wie angezeigt verwendet. Sie entsprechen denen des Diagrammtyps **Daten-Tabelle**.

|**Rohdaten-Export**
|Es wird ebenfalls eine __**CSV-Datei**__ erzeugt. Die Daten werden allerdings detaillierter exportiert. So wird anstelle der Anzahl der Aufträge für jeden Auftrag die relevanten Datenfelder, wie **Auftrags-ID**, **Preis** etc. in die CSV-Datei eingefügt.

|
|Einstellungen

|**Name**
|Die __**Bezeichnung**__ der Statistik, die beim Erstellen gewählt wurde. Der Name kann hier geändert werden.

|**Lager**
|Das __**Lager**__ wählen, das für die Statistik berücksichtigt werden soll. **Alle** wählen, um alle Lager einzubeziehen.

|**Preisauswahl**
|**Durchschnittlicher EK** = Der vom System berechnete __**durchschnittliche Einkaufspreis**__ wird berücksichtigt. +
**Aktueller EK** = Der jeweils __**aktuelle Einkaufspreis**__, der in den Artikeldaten hinterlegt ist, wird für die Statistik herangezogen. +
**Durchschnittlicher EK, wenn 0, dann aktuellen EK verwenden** = Wenn __**kein durchschnittlicher EK hinterlegt**__ ist, wird automatisch der __**aktuelle EK**__ berücksichtigt. +
**Verkaufspreis** = Der Preis, der bei der Option **Preis** des jeweiligen Artikels hinterlegt ist.

|**Einschränkung**
|Einen Status Ihres Warenbestandes wählen, z.B. **Meldebstand unterschritten**. Nur Bestände mit diesem Status werden ausgewertet. **ALLE** wählen, um alle Status einzubeziehen.

|**Markierung 1 und 2**
|Einstellen, ob **alle Artikel**, nur Artikel mit einer **bestimmten Markierung** oder nur Artikel **ohne Markierung** in der Statistik berücksichtigt werden sollen. Standardmäßig sind **alle Artikel** eingestellt. Es können höchstens zwei Markierungen auf einmal gewählt werden.

|**Nicht aktive Artikel**
|Häkchen setzen, um inaktive Artikel auszuwerten. +
Ein Artikel ist inaktiv, wenn im Tab **Verfügbar** ein Häkchen neben **Inaktiv** gesetzt wurde.

|**Paket**
|Häkchen setzen, um Artikelpakete auszuwerten.
|====

__Tab. 1: **Statistik bearbeiten**; **Einstellungen**__

== Daten-Export

Wie in Tabelle 1 erläutert können Sie zwischen einem einfachen und einem detaillierten Exportergebnis wählen. Nachfolgend finden Sie entsprechende Beispiele.

=== Export der Diagrammdaten

[IMPORTANT]
.Datenfelder
====
**ItemID** = Artikel-ID, **PriceID** = Preis-ID des Artikels, **WarehouseID** = Lager-ID, **AttributeValueSetID** = ID der Artikelvariante, **ItemName** = Artikel-Name, **VariationName** = Name der Artikelvariante, **WarehouseName** = Name des Lagers, **PhysicalStock** = Warenbestand gesamt, **NetStock** = Verfügbarer Warenbestand, **ReservedStock** = Reservierter Warenbestand, **ValueOfNetStock** = Wert des verfügbaren Warenbestands, **ValueOfPhysicalStock** = Wert des gesamten Warenbestands, **ValueOfReservedStock** = Wert des reservierten Warenbestands
====
