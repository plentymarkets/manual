== Retourengründe

Diese Statistik zeigt die Beträge der __**Retouren**__ an, in Abhängigkeit der __**Zeitspanne**__ und nach __**Retourengrund**__. Alternativ zu einer Zeitspanne können Sie den Zeitraum per **Start**- und **Enddatum** eingrenzen. Standardeinstellung der __**Zeitspanne**__: die **letzten 10 Tage**.

== Statistik-Details

Entsprechend der gewählten Zeitspanne wird die jeweilige __**Retourenanzahl**__ und der jeweilige __**Umsatz in Euro**__ angezeigt, außerdem sind die Retouren nach __**Retouregründen**__ sortiert. Sind keine Retouren für einen Retourengrund vorhanden, wird der jeweilige Retourengrund nicht angezeigt. Als Tooltip wird zu einem __**Retouregrund**__ der zugehörige __**Betrag in Euro**__ angezeigt (Bild 1). Sie können mit dieser Statistik z.B. erkennen, wieviele Retouren vorhanden sind und wieviel Umsatz im jeweiligen Zeitraum durch Retouren betroffen ist. Außerdem können Sie ablesen, wie Anzahl und Umsatz sich auf die Retourengründe verteilen. Das dient dem Controlling Ihrer Rückläufer.

In Bild 1 wird z.B. zum Retourengrund __**defekt**__ ein Betrag von __**393,49 Euro**__ für __**3 Retouren**__ angezeigt.

== Einstellungen

Legen Sie zunächst wie auf der Handbuchseite <<basics/statistik#, Statistik>> beschrieben eine neue Statistik an. Für die Statistik **Retourengründe** stehen dann die in Tabelle 1 beschriebenen __**Einstellungen**__ zur Verfügung.

__Bild 2: Statistik **Retourengründe**; **Einstellungen**__

Die folgende Tabelle erläutert die Einstellungen im Menü **Start » Statistik → Statistik-Typ: Aufträge / Retourengründe**:

[cols="a,a"]
|====
|Einstellung |Erläuterung

|
|Diagramm-Anzeige

|**Diagrammtyp**
|**Säulen- und Liniendiagramm** = Darstellung der Werte als Säule bzw. Linie +
**2D-Kreisdiagramm** = Darstellung der Werte als flächige Kreissektoren +
**3D-Kreisdiagramm** = Darstellung der Werte als dreidimensionale Kreissektoren +
**Daten-Tabelle** = Datenübersicht in Tabellenform

|**Export**
|Die Daten des Diagramms werden in eine __**CSV-Datei**__ exportiert. Dabei werden die Parameter wie angezeigt verwendet. Sie entsprechen denen des Diagrammtyps **Daten-Tabelle**.

|**Rohdaten-Export**
|Es wird ebenfalls eine __**CSV-Datei**__ erzeugt. Die Daten werden allerdings detaillierter exportiert. So wird anstelle der Anzahl der Aufträge für jeden Auftrag die relevanten Datenfelder, wie **Auftrags-ID**, **Preis** etc. in die CSV-Datei eingefügt.

|
|Einstellungen

|**Name**
|Die __**Bezeichnung**__ der Statistik, die beim Erstellen gewählt wurde. Der Name kann hier geändert werden.

|**Zeitspanne**
|Eine Zeitspanne für die Statistik aus dem Dropdown-Menü wählen. Standardwert: **Die letzten 10 Tage**. +
Die Zeitspanne ändern, wenn keine oder nur wenige Daten angezeigt werden. Dann auf **Speichern** klicken, um das Ergebnis anzuzeigen. +
Durch Wahl eines __**längeren Zeitraums**__ werden mehr Informationen angezeigt. Bei vielen Daten ermöglicht die __**Verringerung der Zeitspanne**__ ggf. eine aussagekräftigere Auswertung.

|**Startdatum / Enddatum**
|Optional einen bestimmten __**Datumsbereich**__ alternativ zur Option **Zeitspanne** eingrenzen.

|**Von Status / Bis Status**
|Je nach Statistiktyp stehen zur Eingrenzung des **Status** verschiedene Optionen zur Verfügung. Es sind jeweils __**Standardwerte**__ voreingestellt.

|**Mandant (Shop)**
|Die __**Webshops**__ wählen, deren Daten für die Statistik verwendet werden sollen.

|**Auftragsherkunft**
|Die __**Auftragsherkünfte**__ wählen, deren Daten für die Statistik verwendet werden sollen.

|**Diagramm**
|Wählen, ob __**Umsätze**__ in der Statistik als __**Brutto**__ oder __**Netto**__ angezeigt werden sollen.

|**Amazon-Konto**
|Das __**Amazon-Konto**__ wählen, das für die Statistik berücksichtigt werden soll.

|**eBay-Konto**
|Das __**eBay-Konto**__ wählen, das für die Statistik berücksichtigt werden soll.
|====

== Daten-Export

Wie in Tabelle 1 erläutert können Sie zwischen einem einfachen und einem detaillierten Exportergebnis wählen. Nachfolgend finden Sie entsprechende Beispiele.

.Datenfelder
====
**ReasonOfReturnID** = ID des Retouregrundes, **ReasonOfReturnName** = Bezeichnung des Retouregrundes, **Sales** = Umsatz, **SalesNet** = Umsatz netto, **QuantityOfOrders** = Anzahl der Aufträge, **ReturnedItems** = Anzahl der retournierten Artikel
