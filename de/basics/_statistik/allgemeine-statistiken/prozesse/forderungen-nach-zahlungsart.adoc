== Forderungen nach Zahlungsart

Offene Zahlungen werden nach __**Zahlungsart**__ und __**offener Forderung mit Mahnstufe (0 bis 4)**__ angezeigt. Der Auftragsstatus, der berücksichtigt werden soll, ist genau eingrenzbar. Standardeinstellung des __**Statusbereichs**__: Von **Status 1** bis **Status 7**.

== Statistik-Details

Entsprechend des gewählten Bereichs des __**Auftragsstatus**__ wird die __**Summe der offenen Forderungen pro Zahlungsart und Mahnstufe**__ angezeigt. Sind keine offenen Forderungen vorhanden, wird die entsprechende Zahlungsart nicht angezeigt. Als Tooltip wird der Gesamtbetrag der offenen Forderungen einer Zahlungsart für jede Mahnstufe in einer anderen Farbe angezeigt (Bild 1). Sie können mit dieser Statistik z.B. erkennen, bei welchen Zahlungsarten die meisten offenen Forderungen und Mahnstufen auftreten.

In Bild 1 wird z.B. für die Zahlungsart __**vorab Überweisung / Vorkasse**__ eine offene Forderung von __**3356,32 Euro**__ für die __**Mahnstufe 0**__ angezeigt.

== Einstellungen

Legen Sie zunächst wie auf der Handbuchseite <<basics/statistik#, Statistik>> beschrieben eine neue Statistik an. Für die Statistik **Forderungen nach Zahlungsart** stehen dann die in Tabelle 1 beschriebenen __**Einstellungen**__ zur Verfügung.

image::basics/statistik/allgemeine-statistiken/prozesse/assets/DE-Statistik-OffeneForderungen-02-SI.png[]

__Bild 2: Statistik **Forderungen nach Zahlungsart**; **Einstellungen**__

Die folgende Tabelle erläutert die Einstellungen im Menü **Start » Statistik → Statistik-Typ: Prozesse / Forderungen nach Zahlungsart**:

[cols="a,a"]
|====
|Einstellung |Erläuterung

|
|Diagramm-Anzeige

|**Diagrammtyp**
|**Gestapeltes-Balkendiagramm** = Darstellung der Werte als Balken. Die offenen Forderungen für die **Mahnstufe 0** (blau) werden ganz links angezeigt. Danach folgen die offenen Forderungen für die **Mahnstufe 1** (rot), die offenen Forderungen für die **Mahnstufe 2** (grün) und schließlich die offenen Forderungen für die **Mahnstufe 3** (gelb) und die **Mahnstufe 4** (hellblau). +
**Daten-Tabelle** = Datenübersicht in Tabellenform

|**Export**
|Die Daten des Diagramms werden in eine __**CSV-Datei**__ exportiert. Dabei werden die Parameter wie angezeigt verwendet. Sie entsprechen denen des Diagrammtyps **Daten-Tabelle**.

|**Rohdaten-Export**
|Es wird ebenfalls eine __**CSV-Datei**__ erzeugt. Die Daten werden allerdings detaillierter exportiert. So wird anstelle der Anzahl der Aufträge für jeden Auftrag die relevanten Datenfelder, wie **Auftrags-ID**, **Preis** etc. in die CSV-Datei eingefügt.

|
|Einstellungen

|**Name**
|Die __**Bezeichnung**__ der Statistik, die beim Erstellen gewählt wurde. Der Name kann hier geändert werden.

|**Von Status / Bis Status**
|Je nach Statistiktyp stehen zur Eingrenzung des **Status** verschiedene Optionen zur Verfügung. Es sind jeweils __**Standardwerte**__ voreingestellt.

|**Mandant (Shop)**
|Die __**Webshops**__ wählen, deren Daten für die Statistik verwendet werden sollen.

|**Auftragsherkunft**
|Die __**Auftragsherkünfte**__ wählen, deren Daten für die Statistik verwendet werden sollen.

|**Versandprofil**
|Das __**Versandprofil**__ wählen, das für die Statistik berücksichtigt werden soll. **Alle** wählen, um alle Versandprofile einzubeziehen.

|**Nur bezahlte Aufträge**
|Häkchen setzen, um __**nur bezahlte Aufträge**__ in die Statistik einzubeziehen.

|**Amazon-Konto**
|Das __**Amazon-Konto**__ wählen, das für die Statistik berücksichtigt werden soll.

|**eBay-Konto**
|Das __**eBay-Konto**__ wählen, das für die Statistik berücksichtigt werden soll.
|====

__Tab. 1: **Statistik bearbeiten**; **Einstellungen**__

== Daten-Export

Wie in Tabelle 1 erläutert können Sie zwischen einem einfachen und einem detaillierten Exportergebnis wählen. Nachfolgend finden Sie entsprechende Beispiele.

=== Export der Diagrammdaten

[IMPORTANT]
.Datenfelder
====
**PaymentMethodName** = Name der Zahlungsart, **QuantityOfOrders** = Anzahl der Aufträge, **Claim** = Offene Forderung, **DunningLevel** = Mahnstufe, **PaymentMethodID** = ID der Zahlungsart
====
