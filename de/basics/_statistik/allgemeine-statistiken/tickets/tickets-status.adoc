== Tickets nach Status

Auswertung des **Tickets nach Status** gemäß des gewählten **Ticket-Typs** innerhalb einer gewählten Zeitspanne. Standardeinstellung der Zeitspanne: **Die letzten 6 Monate**.

== Statistik-Details

Es wird die **Anzahl der Tickets****statusbezogen** und der **Zeitraum** angezeigt. Als **Tooltip** wird die Ticketanzahl und der jeweils zugehörige **Status** angezeigt, außerdem in einer anderen Farbe der genaue **Zeitraum**, in dem die jeweiligen Tickets eingegangen sind oder bearbeitet wurden (siehe Bild 1). Wenn zu einem Status keine Tickets vorhanden sind, wird der jeweilige Status nicht angezeigt. Sie können mit dieser Statistik z.B. erkennen, wie hoch die **Anzahl der Tickets eines Typs** im jeweiligen Status ist.

[NOTE]
.**Details anzeigen (Tooltip)**
====
Zeigen Sie mit dem **Mauszeiger** auf ein Element der Statistik, um die jeweiligen **Details** anzuzeigen.
====

[TIP]
.Tipp
====
In Bild 1 wird z.B. für den Status **Neu** eine Ticketanzahl von **624** angezeigt, die im Zeitraum **2.2013** eingingen.
====

== Einstellungen

Die nachfolgend beschriebenen **Einstellungen** stehen für diese Statistik zur Verfügung.

image::basics/statistik/allgemeine-statistiken/tickets/assets/DE-Statistik-Tickets-TicketsStatus-02-SI.png[]

__Bild 2: Statistik **Tickets nach Status**; **Einstellungen**__

Beachten Sie die Erläuterungen in der folgenden Tabelle:

[cols="a,a"]
|====
|Einstellung |Erläuterung

|
|Diagramm-Anzeige

|**Diagrammtyp**
|In diesem Dropdown-Menü wählen Sie die Art der Darstellung aus. +
**Multi-Liniendiagramm** = Darstellung der Werte als Linien +
**Daten-Tabelle** = Datenübersicht in Tabellenform

|**Export**
|Die Daten des Diagramms werden in eine **CSV-Datei** exportiert. Dabei werden die Parameter wie angezeigt verwendet, sie entsprechen denen des Diagrammtyps **Daten-Tabelle**.

|
|Einstellungen

|**Name**
|Hier wird der **Name** bzw. die **Bezeichnung** der Statistik angezeigt, die beim Erstellen einer neuen Statistik gewählt wurde. Sie können diesen Namen hier ändern.

|**Zeitspanne**
|Die **Zeitspanne** ist für die Statistiktypen jeweils voreingestellt, die Standardwerte können Sie Tabelle 1 der Handbuchseite <<basics/statistik#, Statistik>> entnehmen. Es stehen feste Werte zur Verfügung, die per Dropdown-Menü auswählbar sind. Darüber hinaus bieten manche Statistiktypen die Möglichkeit, eine Zeitspanne in Form eines **Datumbereichs** einzutragen. +
**Verändern** Sie die **Zeitspanne**, wenn keine Daten angezeigt werden oder nur wenige Daten und klicken Sie auf das Speichern-Symbol, um das Ergebnis zu sehen. Sie können durch Wahl eines **längeren Zeitraums** mehr Informationen anzeigen. Wenn viele Daten vorhanden sind, bewirkt die **Verringerung der Zeitspanne** ggf. eine aussagekräftigere Auswertung.

|**Startdatum ... Enddatum**
|Hier können Sie optional einen bestimmten **Datumsbereich** alternativ zur Option **Zeitspanne** eingrenzen.

|**Status**
|Hier stehen Ihnen die von Ihnen angelegten **Ticket-Typen**, mit den dazugehörigen **Status** für die Statistik zur Auswahl. Standardmäßig sind alle Typen und Status ausgewählt.

|**Kunden-ID**
|Geben Sie hier eine **Kunden-ID** ein, um sich eine Statistik der Ticketdaten dieses **einzelnen Kunden** anzeigen zu lassen.

|**Kundenklasse**
|Wählen Sie die **Kundenklasse(n)**, deren Daten für die Statistik verwendet werden sollen.

|**Priorität**
|Wählen Sie die **Ticket-Priorität(en)**, die in der Statistik berücksichtigt werden sollen.
|====
